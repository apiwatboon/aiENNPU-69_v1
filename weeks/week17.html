AI สร้างภาพได้อย่างไร: Diffusion & Multimodal

หลักการเดิม (ทำนาย) — แต่คราวนี้ทำนาย "noise ที่ต้องลบออก"

🎯 เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายหลักการ diffusion ได้: เทรนให้ "ลบ noise ทีละนิด" แล้วสร้างภาพจาก noise ล้วน
  2. อธิบายได้ว่า text prompt ควบคุมภาพที่ออกมาได้อย่างไร (ผ่าน embedding ร่วม)
  3. เห็นว่าความรู้ทั้งเทอม (loss, เทรน, สถาปัตยกรรม) ใช้กับ AI ต่างชนิดได้หมด

ส่วนที่ 1 — แนวคิด แนวคิด

คำถามเปิดคาบ: "จะสอนใครสักคน 'วาดภาพจากศูนย์' กับสอนให้ 'ค่อยๆ ลบรอยเปื้อนออกจากภาพ' — อะไรสอนง่ายกว่า?"

Diffusion: เรียนรู้การ "กู้ภาพคืน" จาก noise

แทนที่จะสอนโมเดลวาดภาพตรงๆ (ยากมาก) เราพลิกโจทย์:

  1. ขั้นเทรน: เอาภาพจริง → เติม noise ทีละขั้นจนกลายเป็นจุดสุ่มล้วน แล้วเทรนโมเดลให้ทำงานเดียว: "ทายว่า noise ที่เพิ่งถูกเติมเข้าไปคืออะไร" — นี่คือโจทย์ supervised learning ธรรมดา! มี input (ภาพเปื้อน) มีเฉลย (noise ที่เราเติมเอง) มี loss (ทาย noise ผิดแค่ไหน) เทรนด้วย backprop ตามเคย
  2. ขั้นสร้างภาพ: เริ่มจาก noise สุ่มล้วนๆ → ให้โมเดลทาย noise แล้วลบออก ทีละขั้น หลายสิบรอบ → ภาพ "โผล่" ออกมาจากหมอก

เทียบกับ LLM: LLM ทำนาย "token ถัดไป" ทีละตัว — Diffusion ทำนาย "noise ที่ต้องลบ" ทีละขั้น ต่างกันแค่ว่าทำนายอะไร โครงคิดเดียวกันหมด

Text prompt คุมภาพได้อย่างไร

โมเดลอย่าง CLIP เรียน "แผนที่ความหมายร่วม" ระหว่างข้อความกับภาพ (ภาพแมวกับคำว่า "cat" อยู่ใกล้กันในแผนที่เวกเตอร์) ตอน diffusion ลบ noise แต่ละขั้น เราเอาเวกเตอร์ของ prompt ไป "ดึงทิศ" ให้การลบ noise เอียงไปทางภาพที่ตรงความหมาย — attention ระหว่างคำกับบริเวณภาพอีกแล้ว!

Multimodal: ทุกอย่างคือ token

โมเดลยุคล่าสุด (GPT-4o, Claude, Gemini) มองภาพ/เสียง/ข้อความเป็น "token" ในกระแสเดียวกัน — ภาพถูกหั่นเป็นชิ้นเล็กแปลงเป็นเวกเตอร์เข้า Transformer ตัวเดียวกับข้อความ บทเรียนเรื่อง CNN กลับมาอีกครั้ง: ออกแบบ "วิธีป้อนข้อมูล" ให้เข้ากับสถาปัตยกรรม

👣 เข้าใจทีละขั้น — AI วาดรูปได้อย่างไร ใน 6 ขั้น

แบ่งเป็นสองเฟส: ตอนเรียน (ขั้น 1–2) กับตอนวาดจริง (ขั้น 3–5) — แล้วปิดด้วยวิธีที่ข้อความคุมภาพ

  1. เฟสเรียน: สร้างโจทย์ฟรีไม่จำกัดข้อ เอาภาพแมวจริง 1 ใบ เติม noise เข้าไป 10% — ตอนนี้เรามี "โจทย์พร้อมเฉลย" หนึ่งข้อ: input = ภาพเปื้อน 10%, เฉลย = noise ที่เราเพิ่งเติมเอง (เรารู้เป๊ะเพราะเราเติม!)
    ทำซ้ำกับทุกระดับ noise (1%–100%) กับภาพหลายร้อยล้านใบ → ได้โจทย์ไม่จำกัดข้อ ฟรี ไม่ต้องจ้างใครเฉลย
  2. เทรนด้วยเครื่องมือเดิมทุกชิ้น โมเดลทาย noise → เทียบกับเฉลย → loss คือ MSE (เรื่อง Polynomial Regression!) → backprop (เรื่อง Backpropagation)
    จบเฟสเรียน: ได้เซียนที่เก่งงานเดียว — "ชี้ noise ในภาพเปื้อนได้แม่นมาก" แค่นั้นจริงๆ
  3. เฟสวาด: เริ่มจากความว่างเปล่าสุดขั้ว สุ่ม noise ล้วน 100% หนึ่งผืน (เหมือนทีวีจอซ่า) — ยังไม่มีภาพอะไรอยู่ในนั้นเลย
  4. ถามเซียนซ้ำๆ: "noise ตรงไหน" แล้วลบออกทีละนิด เซียนชี้ noise → ลบออกส่วนหนึ่ง → ภาพชัดขึ้นนิดหนึ่ง → ถามใหม่ → ลบอีกนิด...
    ทำราว 20–50 รอบ ภาพ "โผล่จากหมอก": โครงใหญ่ (ฟ้า/พื้น) มาก่อน รายละเอียด (หนวด ขน) มาท้าย
    กดปุ่ม ▶ ใน demo ข้างล่าง ดูภาพโผล่จาก noise — สังเกตว่าอะไรชัดก่อน-หลัง ตรงกับที่บอกไหม
  5. ทำไมแต่ละครั้งได้ภาพไม่ซ้ำ เพราะขั้น 3 เริ่มจาก noise สุ่มคนละผืน — เซียนลบ noise ไปคนละทิศ ภาพปลายทางจึงต่างกัน (เหมือน LLM ที่สุ่มจาก softmax — ความสุ่มอยู่คนละจุด แต่บทบาทเดียวกัน)
  6. ใส่ prompt: ดึงทิศการลบ noise ข้อความ "แมวส้มใส่หมวก" ถูกแปลงเป็นเวกเตอร์ความหมาย (CLIP) แล้วป้อนให้เซียนทุกรอบที่ลบ noise — การลบจึงเอียงไปทาง "ภาพที่ความหมายตรงกับข้อความ" ทีละนิดจนภาพสุดท้ายตรง prompt
    สรุปทั้งระบบ: ทำนาย (noise) + loss + backprop + embedding + attention — ไม่มีของใหม่ มีแต่ของเก่าพลิกมุม
    ไปทำการทดลอง 3 ข้อใน demo 🧪 ทดลองกับเครื่องมือจริง (ท้ายส่วนที่ 2) แล้วอธิบายผลแต่ละข้อด้วยขั้น 3–6 ข้างบน

🧩 ตัวอย่างหลายแบบ — diffusion ไม่ได้มีแค่ภาพ

ตัวอย่าง 1 — คำนวณภาพเปื้อนด้วยมือ

pixel ภาพจริงมีค่าความสว่าง 200, noise ที่สุ่มได้คือ 80 — ที่ระดับ noise 30% pixel ภาพเปื้อนมีค่าเท่าไหร่? (สูตรผสม: เปื้อน = จริง × (1−ระดับ) + noise × ระดับ)

ขั้น 1 — ส่วนของภาพจริง: 200 × (1−0.3) = 200 × 0.7 = 140
ขั้น 2 — ส่วนของ noise: 80 × 0.3 = 24
ขั้น 3 — รวม: 140 + 24 = 164 (ลองที่ 90%: 200×0.1 + 80×0.9 = 92 — แทบเป็น noise แล้ว)
✅ คำตอบ: 164 — สไลเดอร์ใน demo รันสูตรนี้กับทุก pixel พร้อมกัน และคู่ (ภาพเปื้อน 164, noise 80) นี่แหละคือ "โจทย์พร้อมเฉลย" ที่ใช้เทรนโมเดลจริง
ตัวอย่าง 2 — เพลงและวิดีโอ: กลไกเดิม เปลี่ยนแค่ข้อมูล

AI แต่งเพลง (เช่น Suno) กับ AI ทำวิดีโอ (เช่น Sora) ใช้หลักอะไร — ลองเดาจากที่เรียนก่อนกางเฉลย

ขั้น 1 — ตั้งคำถามนำ: "ข้อมูลชนิดนี้ แปลงเป็นรูปแบบที่ diffusion กินได้ยังไง?"
ขั้น 2 — เพลง: เสียงแปลงเป็น "ภาพคลื่นความถี่" (spectrogram: แกนนอนเวลา แกนตั้งความถี่) → ทำ diffusion บน "ภาพ" นั้น → แปลงกลับเป็นเสียง
ขั้น 3 — วิดีโอ: ลบ noise หลายเฟรมพร้อมกัน โดยบังคับเงื่อนไขให้เฟรมติดกันต่อเนื่อง
✅ คำตอบ: ตระกูลเดียวกันทั้งคู่ — บทเรียนเรื่อง CNN ทำงานอีกครั้ง: เปลี่ยน "วิธีป้อนข้อมูล" ให้เข้าเครื่องจักรเดิม เจอ AI สร้างคอนเทนต์ชนิดใหม่เมื่อไหร่ เดาก่อนเลยว่าข้างในคือ diffusion หรือ next-token
ตัวอย่าง 3 — ปุ่ม CFG scale: เชื่อ prompt แค่ไหนดี

เครื่องมือสร้างภาพหลายตัวมีปุ่ม "guidance/CFG scale" — ถ้าตั้งต่ำมาก (1–2) กับสูงมาก (15–20) ภาพจะต่างกันยังไง เพราะอะไร?

ขั้น 1 — ทวนหน้าที่ของ CFG scale: มันคือตัวคูณ "แรงดึงทิศของ prompt" ตอนลบ noise แต่ละขั้น
ขั้น 2 — กรณี scale ต่ำ (1–2): แรงดึงเบา → โมเดลลบ noise ตามธรรมชาติของภาพ → สวย หลากหลาย แต่อาจหลุดธีมจาก prompt
ขั้น 3 — กรณี scale สูง (15–20): แรงดึงรุนแรงทุกขั้น → ตรง prompt เป๊ะ แต่ภาพถูกบิดตามแรงดึง: สีจัด คอนทราสต์แข็ง ดูปลอม
✅ คำตอบ: ต่ำ=สวยแต่เฉไฉ สูง=ตรงแต่แข็ง — ค่ามาตรฐานที่นิยมคือ 7–8 (สมดุล) ลองลากปรับเองในเดโม 🎚️ CFG ส่วนที่ 2 แล้วต่อด้วยเครื่องมือจริง
ตัวอย่าง 4 — คำนวณย้อนกลับ: กู้ pixel จริงคืน

จากตัวอย่าง 1: pixel เปื้อนค่า 164 (ที่ noise 30%) — ถ้าโมเดลทาย noise ได้แม่นเป๊ะว่าคือ 80 จะกู้ค่าจริงคืนได้เท่าไหร่? (สูตร: จริง = (เปื้อน − 0.3×noise) ÷ 0.7)

ขั้น 1 — หักส่วนของ noise ออก: 164 − 0.3×80 = 164 − 24 = 140
ขั้น 2 — ขยายกลับเต็มสเกล: 140 ÷ 0.7 = 200
ขั้น 3 — ตรวจ: ตรงค่าจริง (200) เป๊ะ — เพราะเราสมมุติว่าทาย noise แม่น 100%
✅ คำตอบ: 200 — ทั้งกระบวนการสร้างภาพคือสมการชั้นมัธยมนี้ทำซ้ำๆ ความเก่งทั้งหมดอยู่ที่ "ทาย noise ให้แม่น" ซึ่งคือหน้าที่ของ neural network
ตัวอย่าง 5 — ทำไมไม่ลบ noise รวดเดียวให้จบ

ในเมื่อโมเดลทาย noise ได้ ทำไมไม่ทายทีเดียวจากภาพ noise 100% แล้วลบครั้งเดียว — ต้องวน 50 รอบให้เสียเวลาทำไม?

ขั้น 1 — ลองลบทีเดียว: ที่ noise 100% ไม่มีเบาะแสอะไรเลย — คำตอบที่ "ปลอดภัยสุด" ของโมเดลคือค่าเฉลี่ยของทุกภาพ = ภาพเบลอๆ ไร้รายละเอียด
ขั้น 2 — ลบทีละนิดแทน: รอบแรกลบนิดเดียว → โครงรางๆ เริ่มโผล่ (ฟ้าอยู่บน พื้นอยู่ล่าง)
ขั้น 3 — รอบถัดไปฉลาดขึ้น: มีโครงเป็นเบาะแส → ทาย noise แม่นขึ้น → ภาพชัดขึ้น → เบาะแสมากขึ้นอีก — วงจรเสริมตัวเอง
✅ คำตอบ: ลบรวดเดียว = ทายมั่วจากความว่างเปล่า | ลบทีละนิด = ทุกขั้นใช้ข้อมูลจากขั้นก่อน — ปรัชญาเดียวกับ gradient descent: ก้าวเล็กๆ ที่ใช้ข้อมูลใหม่ทุกก้าว ชนะการกระโดดครั้งเดียว
ตัวอย่าง 6 — seed: คุมความสุ่มได้

เครื่องมือสร้างภาพมีช่อง "seed" — ตั้ง seed 12345 + prompt เดิม สร้างกี่ครั้งก็ได้ภาพเดิมเป๊ะ — อธิบายด้วยกลไก แล้วบอกว่ามีประโยชน์ตอนไหน

ขั้น 1 — หาว่าความสุ่มอยู่ตรงไหน: ในกระบวนการสร้างภาพ ความสุ่มมีจุดเดียว: noise ตั้งต้น
ขั้น 2 — seed ทำอะไร: ตัวเลขตั้งต้นของเครื่องสุ่ม — seed เดิมให้ "noise ผืนเดิม" เป๊ะทุกครั้ง
ขั้น 3 — ไล่ผล: จุดเริ่มเดิม + prompt เดิม (เส้นทางลบ noise เดิม) = ปลายทางเดิม — ภาพซ้ำได้ 100%
✅ คำตอบ: ใช้จริงสองทาง: (1) ชอบองค์ประกอบแล้วอยากแก้รายละเอียด → ล็อก seed แก้แค่ prompt (2) งานวิจัย/รายงานต้องทำซ้ำได้ → รายงาน seed เสมอ (ใส่ในโปรเจกต์ได้คะแนน)
ตัวอย่าง 7 — Magic Eraser: diffusion เฉพาะที่ (inpainting)

ฟีเจอร์ "ลบคนแปลกหน้าหลังภาพ" ในมือถือ — ใช้กลไกอะไร? ใบ้: มันคือ diffusion ที่จำกัดบริเวณ

ขั้น 1 — ผู้ใช้ระบายทับคนที่จะลบ: บริเวณนั้นถูกแทนด้วย noise 100% (ส่วนที่เหลือของภาพคงเดิม)
ขั้น 2 — denoise เฉพาะที่: โมเดลลบ noise บริเวณนั้น โดยมองเห็นบริบทรอบๆ (พื้นหญ้า กำแพง แสง)
ขั้น 3 — ผลลัพธ์: มันเติมสิ่งที่ "ควรจะอยู่ตรงนั้น" ตามบริบท — พื้นหญ้าต่อเนื่อง เงาสอดคล้อง
✅ คำตอบ: diffusion เฉพาะบริเวณ + attention เชื่อมบริเวณว่างกับรอบข้าง (เรื่อง Attention & Transformer อีกแล้ว) — ฟีเจอร์ดูเวทมนตร์ แต่คุณมีชิ้นส่วนความรู้ครบหมดแล้ว
ตัวอย่าง 8 — ทำไม AI วาดนิ้วมือพัง (เคยพัง)

ยุค 2022–23 ภาพคนจาก AI มักมี 6–7 นิ้ว — อธิบายด้วยธรรมชาติของ diffusion ว่าทำไม "นิ้ว" ยากกว่า "ผิว/ผม"

ขั้น 1 — ดูสิ่งที่ diffusion เก่ง: สถิติเฉพาะที่ — ลายผิว แสงเงา ข้อนิ้ว: มองแพตช์เล็กๆ ทีละจุด ทุกจุดสมจริงหมด
ขั้น 2 — ดูสิ่งที่ "มือ" ต้องการ: กฎเชิงโครงสร้าง "มีนิ้ว 5 นิ้วพอดี" — ต้องนับข้ามทั้งบริเวณ ไม่ใช่ดูทีละจุด
ขั้น 3 — เห็นช่องว่าง: แต่ละแพตช์เหมือนนิ้วจริงเป๊ะ แค่จำนวนรวมเป็น 6 — สถิติเฉพาะที่ผ่านหมด กฎรวมพัง
✅ คำตอบ: เพราะความสมจริงเฉพาะจุด ≠ ความถูกต้องเชิงโครงสร้าง — รุ่นใหม่ดีขึ้นด้วยข้อมูลมือมหาศาล+มุมมองกว้างขึ้น แต่หลักจำ: งาน "นับ/ตรรกะ" คือจุดอ่อนคลาสสิกของโมเดลตระกูลสถิติ (LLM นับตัวอักษรพลาดด้วยเหตุผลพี่น้องกัน — tokenization)
ตัวอย่าง 9 — สไตล์ศิลปิน: คำถามที่ศาลทั่วโลกยังไม่จบ

โมเดลเทรนจากภาพศิลปินจริงนับล้านโดยไม่ขออนุญาต — สั่ง "วาดสไตล์ศิลปิน X" ได้ภาพคล้ายมาก — ใช้กลไกที่เรียนมาอธิบาย: มัน "ก๊อป" หรือ "เรียนรู้"?

ขั้น 1 — ฝั่ง "เรียนรู้": โมเดลไม่ได้เก็บไฟล์ภาพต้นฉบับ — มันเก็บ weight ที่จับ "สถิติของสไตล์" (โทนสี ฝีแปรง องค์ประกอบ) เหมือนคนเรียนวาดจากการดูงานเยอะๆ
ขั้น 2 — ฝั่ง "ก๊อป": ถ้าภาพไหนซ้ำในข้อมูลฝึกบ่อย โมเดล overfit ภาพนั้น (ส.6!) แล้วผลิตใกล้ต้นฉบับได้จริง — มีงานวิจัยสกัดภาพฝึกออกมาได้
ขั้น 3 — ปัจจัยที่เทคนิคตอบไม่ได้: สเกล (ล้านภาพ/วัน) + ความเร็ว + ไม่ขออนุญาต ทำให้ผลกระทบต่อศิลปิน คนละเรื่องกับนักเรียนศิลปะคนหนึ่งฝึกตาม
✅ คำตอบ: เชิงกลไกคือ "เรียนสถิติ" เป็นหลัก แต่ก๊อปได้ในเคส overfit — เทคนิคตอบได้ครึ่งเดียว อีกครึ่งเป็นกฎหมาย/จริยธรรม ซึ่งสัปดาห์หน้าเจาะเต็มคาบ
ตัวอย่าง 10 — จาก 1 ภาพ เป็นวิดีโอ 240 ภาพ: ความยากเพิ่มตรงไหน

วิดีโอ 10 วินาที = 24 เฟรม/วิ × 10 = 240 ภาพ — ถ้าใช้ diffusion สร้างทีละเฟรมอิสระกัน จะได้วิดีโอแบบไหน? แล้วของจริง (Sora ฯลฯ) แก้ยังไง?

ขั้น 1 — ลองวิธีง่าย (สร้างทีละเฟรมอิสระ): แต่ละเฟรมเริ่มจาก noise คนละผืน → ลบ noise คนละเส้นทาง
ขั้น 2 — ดูผล: แต่ละเฟรมสวยเดี่ยวๆ แต่ไม่ต่อเนื่อง: เสื้อเปลี่ยนสี หน้าเปลี่ยนคน ฉากกระพริบ — ดูไม่ได้
ขั้น 3 — วิธีของจริง: denoise ทุกเฟรมพร้อมกัน + attention เชื่อมข้ามเฟรม บังคับ "เสื้อเฟรม 1 ต้องเหมือนเฟรม 240"
✅ คำตอบ: ความยากใหม่คือ "ความต่อเนื่องข้ามเวลา" ซึ่งแลกด้วยงานคำนวณคูณมหาศาล — เหตุที่วิดีโอ AI มาช้ากว่าภาพนิ่ง 2 ปี และแพงกว่ามากจนวันนี้

ส่วนที่ 2 — ทดลองเล่น ทดลองเล่น

🎬 ถอย noise ทีละก้าว — reverse diffusion จริงๆ

นี่คือลูปที่ AI สร้างภาพใช้จริง: มองภาพเลอะ → ทาย noise → ลบออกนิดเดียว → วนใหม่ · ซ้าย = ภาพ ณ เวลา t (t=1000 คือ noise ล้วน, t=0 คือภาพเสร็จ) · ขวา = "คำทาย ε̂" ของโมเดล · กด ▶ ดูครบลูป หรือกด ◀ ▶ เดินเองทีละจังหวะ — ปุ่มลัด t ไว้กระโดดไปช่วงที่สนใจ

ภาพตอนนี้ xt
t = 1000
สิ่งที่โมเดลทาย: noise ε̂
🤝 ตรงไปตรงมา: เดโมนี้ใช้ "โมเดลเก่งสมมติ" ที่ทายแม่นเป๊ะ (เราแอบใช้ noise จริงที่เราเติมเอง) — ของจริงคือ neural network ที่ต้องเทรนให้ทาย (ดูเดโม 🏗️ ข้างล่าง)
กระโดดไป t:
ทำไมต้องถอยทีละก้าว? — ถ้าลบ noise ทั้งหมดในครั้งเดียวจาก t=1000 โมเดลไม่มีเบาะแสอะไรเลย คำตอบที่ "ปลอดภัยสุด" ของมันคือค่าเฉลี่ยของทุกภาพที่เคยเห็น = ก้อนเบลอไร้รายละเอียด · ส่วนการถอยทีละก้าว ทุกก้าวได้เบาะแสใหม่จากก้าวก่อน ภาพจึงคมขึ้นเรื่อยๆ (ตัวอย่าง 5 ข้างบนเป๊ะๆ)
❌ ลบทีเดียว: ได้ "ภาพเฉลี่ย" เบลอๆ
✅ ถอย 11 ก้าว: คมชัดทุกจุด
🔎 ลอง: กด ▶ เล่นจนจบหนึ่งรอบ สังเกตว่า "โครงใหญ่" (ฟ้า/พื้น/ตัวบ้าน) โผล่ก่อน "รายละเอียด" (ประตู ต้นไม้) — แล้วกด ❌ ลบทีเดียวหมด เทียบผลสองฝั่ง: นี่คือคำตอบว่าทำไม diffusion ต้องวนหลายสิบรอบ

🧪 เลื่อนเล่นเอง — forward ↔ reverse

คราวนี้มือเราคือคนคุม t เอง: เลื่อนสไลเดอร์จากขวา (noise 100%) มาซ้าย (noise 0%) — สังเกตว่า "โครงร่างใหญ่" โผล่ก่อน "รายละเอียด" ทีหลัง ตรงกับที่ diffusion จริงสร้างภาพหยาบ→ละเอียด (สูตรผสมเดียวกับเดโม 🎬 ข้างบน — เลื่อนไปขวา = forward เติม noise, เลื่อนกลับซ้าย = reverse)

← ภาพชัด   ระดับ noise:   noise ล้วน →

☁️ มองเป็นฝูงจุด — การกระจายตัวเดินทาง

เปลี่ยนมุมมอง: แทนที่จะดู "ภาพหนึ่งใบ" ลองดู จุดข้อมูล 300 จุดที่เรียงเป็นรูปหน้าแมว · forward = ทุกจุดเดินสุ่มทีละก้าวจนกลายเป็นก้อนหมอก Gaussian · reverse = เดินกลับจนรูปโผล่ — diffusion บนภาพจริงคือสิ่งเดียวกันนี้ แค่ ภาพหนึ่งใบ = จุดหนึ่งจุดในปริภูมิล้านมิติ (พิกเซลละหนึ่งแกน)

t = 0 / 40

🤝 ตรงไปตรงมา: ปุ่ม reverse ในเดโมนี้ "เดินกลับตามเส้นทางเดิมที่จดไว้" เพื่อให้เห็นภาพ — ของจริงโมเดลไม่ได้จำเส้นทางของใคร มันเรียน สนามทิศทาง (score) ที่บอกว่า "จุดที่ตรงนี้ควรขยับไปทางไหน" แล้วเส้นทางกลับทุกเส้นก็งอกจากสนามนั้นเอง

🔎 ลอง: ลากสไลเดอร์ไป-กลับช้าๆ — ช่วง t เล็กรูปยังจำได้ ช่วงท้ายรูปหายเกลี้ยง · เทียบกับเดโมภาพข้างบน: ระดับ noise ของภาพ = ตำแหน่ง t ของฝูงจุดนี่เอง

🏗️ เดินผ่าน U-Net — สถาปนิกของ diffusion

แล้ว "ตัวทาย noise" หน้าตาเป็นยังไงข้างใน? โมเดลมาตรฐานคือ U-Net: ฝั่งซ้าย (encoder) ย่อภาพลงเรื่อยๆ เพื่อจับภาพรวม → ก้นหุบเขา (bottleneck) → ฝั่งขวา (decoder) ขยายกลับเพื่อวาดรายละเอียด โดยมี skip connections ส่ง "ความคม" ข้ามหุบเขา · t และ prompt ถูกฉีดเข้าทุกชั้น — กด ▶ แล้วตามลูกไฟไปทีละชั้น

กด ▶ เพื่อเดินผ่านทีละชั้น — หรือกด ✂️ ตัด skip connections ดูว่าทำไมเส้นส้มถึงสำคัญ
มี skip → ε̂ คมชัด
ตัด skip → เบลอ (ภาพจำลอง)
🔎 ลอง: เดินผ่านจนจบ 1 รอบ แล้วกด ✂️ ตัด skip แล้วเดินใหม่ — สังเกตคำบรรยายชั้น decoder ที่เปลี่ยนไป · โมเดลตัวนี้แหละคือ "เซียนทาย noise" ที่ทำงานอยู่ในเดโม 🎬 ข้อแรก — ถูกเรียกซ้ำทุกก้าว t

🎚️ เชื่อ prompt แค่ไหน — Classifier-Free Guidance

ตอนสร้างภาพ ระบบทาย noise สองครั้ง (ครั้งหนึ่งเห็น prompt ครั้งหนึ่งไม่เห็น) แล้วขยายส่วนต่างด้วยตัวคูณ s — นี่คือปุ่ม cfg_scale ที่อยู่ใน UI ของ Stable Diffusion ทุกตัว · เลือก prompt แล้วลากสไลเดอร์ ดูว่าภาพ "ตัดสินใจ" แรงขึ้นยังไง

prompt:
เป้าหมาย "บ้าน"
เป้าหมาย "หน้าแมว"
guidance s 7.0

🤝 ตรงไปตรงมา: สมการข้างบนคือ CFG จริงจาก deepdive (ส่วนที่ 1) — เดโมนี้ใช้มันกับ "ภาพเป้าหมาย" ตรงๆ แทนที่จะเป็นคำทาย noise ทุกก้าว เพื่อให้เห็นผลทันตา · ที่ s สูงมาก ค่าสีถูกดันทะลุช่วง 0–255 แล้วโดนตัดทิ้ง (clip) — นั่นคือที่มาของภาพ "สีจัด แข็งทื่อ" เวลาตั้ง cfg_scale สูงเกินในเครื่องมือจริง

🔎 ลอง: ตั้ง s=0 (ก้อนเฉลี่ยบ้านปนแมว) → เลื่อนขึ้นช้าๆ หา "จุดที่ภาพตัดสินใจ" → ลากไป 15 ดูสีที่ถูกดันจนเพี้ยน · แล้วสลับ prompt เป็นหน้าแมวโดยไม่ขยับสไลเดอร์ — ตัวอย่าง 3 ข้างบนคือเรื่องเดียวกันนี้ในเครื่องมือจริง

⚡ ทำไม latent diffusion ถึงเร็ว — จับเวลากันจะๆ

การลบ noise ต้องทำ "ทุกช่อง ทุกก้าว" — Stable Diffusion เลยบีบภาพลง latent space เล็กๆ ก่อน (ด้วย autoencoder) แล้วค่อยทำ diffusion ในนั้น เสร็จแล้ว "decode" ขยายกลับเป็นภาพเต็ม · กด ▶ ให้สองฝั่งเริ่มกวาดลบ noise พร้อมกัน ด้วยความเร็วต่อช่องเท่ากันเป๊ะ

pixel space — 48×48 = 2,304 ช่อง

รอเริ่ม — เหลือ 2,304 / 2,304 ช่อง

latent space — 12×12 = 144 ช่อง

รอเริ่ม — เหลือ 144 / 144 ช่อง
⏱ จังหวะที่ 0

🔎 ลอง: สังเกตว่าฝั่ง latent ลบ noise จบแล้ว decode กลับเป็นภาพเต็มตั้งนานแล้ว ฝั่ง pixel ยังกวาดไม่ถึงครึ่ง — งานน้อยกว่ากัน 16 เท่า (144 vs 2,304 ช่อง) นี่คือเคล็ดลับที่ทำให้ Stable Diffusion รันบนการ์ดจอบ้านได้ (ของจริงตาม deepdive: ลดงาน ~50 เท่า)

🧪 ทดลองกับเครื่องมือจริง

ใช้เครื่องมือสร้างภาพฟรีตัวใดก็ได้ (เช่น Bing Image Creator, Stable Diffusion บน Hugging Face) ทดลองเชิงวิทยาศาสตร์:

  1. prompt เดิมเป๊ะ สร้าง 4 ครั้ง — ทำไมได้ภาพไม่ซ้ำ? (เริ่มจาก noise สุ่มคนละผืน!)
  2. เพิ่มรายละเอียด prompt ทีละขั้น: "แมว" → "แมวส้มใส่หมวก" → "แมวส้มใส่หมวกสไตล์สีน้ำ" — สังเกตการ "ดึงทิศ" ที่แรงขึ้น
  3. ลองสั่งสิ่งที่มันมักพลาด: ข้อความบนป้าย, จำนวนนิ้วมือ — แล้วอธิบายว่าทำไมงานแบบนี้ยากสำหรับ diffusion

ส่วนที่ 3 — โค้ดจริง: diffusion ของเล่นใน Colab ลงมือ

diffusion ทั้งระบบเขียนได้ใน ~45 บรรทัด — สคริปต์นี้ทำสองอย่างที่ตรงกับเดโม ☁️ ข้างบนเป๊ะ: (ก) forward process ละลายฝูงจุดรูป "พระจันทร์เสี้ยว 2 อัน" เป็นหมอก (ข) reverse เดินกลับจาก noise จนรูปโผล่ · เปิด Google Colab → New Notebook → วาง → Run — ไม่ต้องติดตั้งอะไรเพิ่ม

import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(42)

# ── (ก) ฝูงจุดรูป "พระจันทร์เสี้ยว 2 อัน" — เหมือนเดโม ☁️ ในบทเรียน ──
n = 600
th = rng.uniform(0, np.pi, n)
x0 = np.c_[np.cos(th), np.sin(th)]                    # เสี้ยวบน
x0[n//2:] = np.c_[1 - np.cos(th[n//2:]), 0.5 - np.sin(th[n//2:])]  # เสี้ยวล่าง
x0 += rng.normal(0, 0.06, x0.shape)

# forward process: x_t = √ᾱ·x₀ + √(1−ᾱ)·ε   (สมการเดียวกับ deepdive)
T = 200
abar = np.cumprod(1 - np.linspace(1e-4, 0.02, T))     # noise schedule

def q_sample(x0, t):
    eps = rng.normal(size=x0.shape)
    return np.sqrt(abar[t]) * x0 + np.sqrt(1 - abar[t]) * eps

fig, ax = plt.subplots(1, 4, figsize=(13, 3.2))
for a, t in zip(ax, [0, 60, 130, 199]):
    xt = x0 if t == 0 else q_sample(x0, t)
    a.scatter(*xt.T, s=4); a.set_title(f"forward: t = {t}")
    a.set_xlim(-3, 4); a.set_ylim(-3, 3.5)
plt.show()                      # รูปค่อยๆ "ละลาย" เป็นก้อน Gaussian

# ── (ข) reverse: เดินกลับจาก noise → รูป ──
# ยอมรับตรงๆ: ของจริงใช้ neural network ที่เทรนให้ทาย noise (แบบ Neural Network + Backpropagation)
# ที่นี่เราแทนโมเดลด้วยตัวทายแบบซื่อๆ: "x₀ ที่น่าจะเป็น = จุดข้อมูลจริงที่ใกล้สุด"
def denoise_step(x, t):
    d = ((x[:, None, :] - x0[None, :, :])**2).sum(-1)   # ระยะถึงจุดจริงทุกจุด
    x0_hat = x0[d.argmin(1)]                            # คำทาย x₀
    if t == 0:
        return x0_hat
    eps_hat = (x - np.sqrt(abar[t]) * x0_hat) / np.sqrt(1 - abar[t])  # ⇒ คำทาย noise
    # ถอยหนึ่งขั้น (สไตล์ DDIM): เล็งไปทาง x₀ แต่คง noise ระดับ t−1 ไว้
    return np.sqrt(abar[t-1]) * x0_hat + np.sqrt(1 - abar[t-1]) * eps_hat

x = rng.normal(size=(n, 2))                             # เริ่มจาก noise ล้วน (t = T)
snap = {}
for t in range(T - 1, -1, -1):
    x = denoise_step(x, t)
    if t in (150, 80, 20, 0): snap[t] = x.copy()

fig, ax = plt.subplots(1, 4, figsize=(13, 3.2))
for a, t in zip(ax, [150, 80, 20, 0]):
    a.scatter(*snap[t].T, s=4, color="tomato"); a.set_title(f"reverse: ถอยถึง t = {t}")
    a.set_xlim(-3, 4); a.set_ylim(-3, 3.5)
plt.show()                      # ฝูงจุดเดินกลับเป็นพระจันทร์เสี้ยว!

  • ภาพแถวบน: ฝูงจุด "ละลาย" เป็นก้อน Gaussian ตามสมการ xt = √ᾱt·x₀ + √(1−ᾱt)·ε — สูตรเดียวกับ deepdive และสไลเดอร์ในเดโมเป๊ะ
  • ภาพแถวล่าง: จุดสีแดงเดินจาก noise กลับเป็นรูปสำเร็จ ทั้งที่ "ตัวทาย" ของเราคือ nearest-neighbor โง่ๆ — พลังอยู่ที่การถอยทีละขั้น ไม่ใช่ความฉลาดของตัวทาย (ของจริงแค่แทนตัวทายด้วย U-Net ที่เทรนแล้ว)
  • ลองเปลี่ยน default_rng(42) เป็นเลขอื่น — ปลายทางยังเป็นพระจันทร์เสี้ยวเหมือนเดิม แต่จุดตกคนละตำแหน่ง: seed คุมความสุ่มทั้งหมด (ตัวอย่าง 6 ข้างบนตรงตัว)

📝 ข้อ 2 (ทำในคาบ) — ทำไมต้อง "พลิกโจทย์"

ทำไมการเทรนให้ "ลบ noise ทีละนิด" ถึงง่ายกว่าเทรนให้ "วาดภาพทั้งใบในครั้งเดียว"? ตอบโดยใช้แนวคิด supervised learning (มี input–เฉลยชัดเจนไหม) และแนวคิดการแบ่งปัญหาใหญ่เป็นขั้นเล็ก

แนวคำตอบ (1) การเติม noise เองทำให้เรามี "คู่ input-เฉลย" ไม่จำกัดฟรีๆ จากภาพจริงทุกใบ — กลายเป็น supervised learning ธรรมดาที่เรารู้วิธีเทรนดี (2) ลบ noise นิดเดียวคือ "ก้าวเล็กที่เรียนได้" เหมือน gradient descent ทีละก้าว — ส่วนการวาดทั้งใบในครั้งเดียวไม่มีสัญญาณบอกว่าผิดตรงไหนควรแก้ทางไหน
🏠 ข้อ 3 (การบ้าน) — รายงานการทดลองสร้างภาพ

ทำการทดลอง 3 ข้อจาก demo 🧪 ทดลองกับเครื่องมือจริง (ท้ายส่วนที่ 2) ส่งภาพประกอบ + รายงาน 1 หน้า โดยทุกข้อสังเกตต้องอธิบายด้วยกลไก (noise เริ่มต้น, การดึงทิศของ prompt, ข้อจำกัดของการมองภาพแบบหยาบ→ละเอียด)

🏠 ข้อ 4 (การบ้าน) — เตรียมหัวข้อโปรเจกต์

ส่งหัวข้อโปรเจกต์ปลายภาค (ดูโจทย์ในหน้าเรื่อง โปรเจกต์ปลายภาค) เป็นย่อหน้าเดียว: จะสร้าง/วิเคราะห์อะไร ใช้แนวคิดจากสัปดาห์ไหนบ้าง — อาจารย์จะ feedback ก่อนเริ่มทำจริง
