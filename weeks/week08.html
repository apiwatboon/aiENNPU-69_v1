Neural Network: สมองจำลอง จาก Perceptron สู่ MLP

70 ปีที่แล้วมนุษย์ลองเลียนแบบเซลล์ประสาทด้วยคณิตศาสตร์ง่ายๆ "คูณ บวก แล้วตัดสินใจ" — วันนี้สิ่งเดียวกันนั้นซ้อนกันพันชั้นอยู่ข้างใน ChatGPT · สัปดาห์นี้เราจะสร้างมันด้วยมือ เห็นตัวเลขวิ่งผ่านทุกโหนด

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. เล่าประวัติย่อของ Neural Network ได้: จากนิวรอนตรรกะ 1943 → Perceptron → AI Winter → Backprop → Deep Learning → LLM
  2. คำนวณ นิวรอนหนึ่งตัว ด้วยมือได้: z = w₁x₁ + w₂x₂ + b แล้วผ่าน activation function (step / sigmoid / ReLU)
  3. ไล่ forward pass ของเครือข่ายหลายชั้น (MLP) ได้ทีละชั้น และแปลงคะแนนชั้นสุดท้ายเป็นความน่าจะเป็นด้วย softmax
  4. อธิบายได้ว่าทำไม ชั้นเดียวแก้ XOR ไม่ได้ แต่เพิ่ม hidden layer แล้วแก้ได้ — เหตุผลที่ต้อง "ลึก"
  5. เชื่อมโยงได้ว่า MLP + softmax คือส่วนประกอบจริงข้างใน Transformer และวิธีที่ LLM "เลือกคำถัดไป"

ส่วนที่ 1 — แนวคิด: เลียนแบบสมองด้วย "คูณ บวก ตัดสินใจ" แนวคิด

เซลล์ประสาทจริงในสมองรับสัญญาณไฟฟ้าจากเพื่อนหลายพันเซลล์ — สัญญาณบางเส้น "ดัง" บางเส้น "เบา" — ถ้ารวมกันแรงพอ เซลล์จะ "ยิง" สัญญาณต่อ นักคณิตศาสตร์มองแล้วสรุปว่า: นี่มันแค่ การบวกแบบถ่วงน้ำหนัก แล้วเทียบเกณฑ์ นี่นา!

z = w₁x₁ + w₂x₂ + … + b   →   a = f(z)

w (weight) = ความดังของแต่ละสาย · b (bias) = ความง่ายในการยิง · f (activation) = กติกาตัดสินใจว่ายิงหรือไม่/แรงแค่ไหน — นิวรอนเทียม 1 ตัวคือแค่นี้จริงๆ ความมหัศจรรย์ทั้งหมดเกิดตอนเอามันต่อกันเป็นชั้นๆ (เรื่อง Linear Regression เราหา w ให้เส้นตรง — วันนี้ w แบบเดียวกันเป๊ะ แค่มีเยอะกว่า)

แต่กว่าจะมาถึงวันนี้ เส้นทางของไอเดียนี้ล้มลุกคลุกคลานถึงขั้น "โดนแช่แข็ง" มาแล้ว — คลิกไล่ดูทีละเหตุการณ์:

ประเด็นสำคัญ: sigmoid/tanh "แบน" ที่ปลายสองข้าง — ความชันเกือบศูนย์ = สัญญาณการเรียนรู้จางหาย (vanishing gradient จะเจอเต็มๆ เรื่อง Backpropagation) · ReLU ง่ายสุดแต่กลายเป็นมาตรฐานของ deep learning เพราะความชันไม่จางฝั่งขวา

⚡ MLP Playground 2-3-2 — ใส่ค่าเอง แก้ weight เอง แล้วดูตัวเลขวิ่งทีละชั้น

เครือข่ายจริงย่อส่วน: input 2 → hidden 3 → output 2 → softmax · โหนดแต่ละตัวผ่าครึ่งด้วยขีดเส้นแนวตั้งตามตำรา: ซีกซ้าย = net (ผลรวมถ่วงน้ำหนัก เช่น h1_net) · ซีกขวา = out (หลังผ่าน activation เช่น h1_out) · เลือก activation ได้ — สลับ ReLU ↔ sigmoid แล้วดูว่า h1_out เปลี่ยนเป็นค่าอะไรจาก net เดิม · กด ▶ Forward ทีละชั้น เห็นค่าปรากฏจากซ้ายไปขวา · เส้นหนา = weight ใหญ่ · น้ำเงิน = บวก / แดง = ลบ · weight ทุกตัวแก้ได้ในตารางข้างล่าง — ค่าเริ่มต้นคือชุดที่แก้โจทย์ XOR ได้จริง!

ส่วนที่ 3 — Forward pass ทำงานทีละขั้น ทีละขั้น

นี่คือสิ่งที่เกิดใน playground ข้างบน (และใน ChatGPT ทุกครั้งที่มันตอบ) — ไล่ตามใน playground ได้ทีละปุ่มกด:

  1. รับ input: ข้อมูลถูกแปลงเป็นตัวเลขเวกเตอร์ x = (x₁, x₂) — ในโลกจริงอาจเป็นพิกเซลภาพ 784 ช่อง หรือ embedding ของคำ
  2. ชั้นซ่อนคำนวณ z: โหนดซ่อนแต่ละตัวคูณ-บวก: zⱼ = wⱼ₁x₁ + wⱼ₂x₂ + bⱼ — สามโหนดก็สามสมการ (นี่คือการคูณเมทริกซ์ W⁽¹⁾x + b⁽¹⁾ นั่นเอง)
  3. ผ่าน activation: aⱼ = ReLU(zⱼ) = max(0, zⱼ) — ตัดค่าลบทิ้ง เกิดความไม่เชิงเส้นที่ทำให้เครือข่าย "ดัดรูป" ปริภูมิได้
  4. ชั้นออกคำนวณคะแนนดิบ (logits): z_out = W⁽²⁾a + b⁽²⁾ — คลาสละหนึ่งคะแนน ยังเป็นเลขอะไรก็ได้ บวกลบได้หมด
  5. softmax แปลงเป็นความน่าจะเป็น: p_i = e^{z_i} / Σe^{z_j} — บวกทุกตัว รวมได้ 1 อ่านเป็น "ความมั่นใจ" ได้ทันที
  6. ตัดสินใจ: เลือกคลาสที่ p สูงสุด (หรือสุ่มตาม p — แบบที่ LLM เลือกคำถัดไป) — จบหนึ่ง forward pass เต็มๆ

ส่วนที่ 4 — ตัวอย่างคำนวณด้วยมือ (เลขเดียวกับ playground) ลงมือ

ตัวอย่าง A — forward pass เครือข่าย 2-2-1: input (1, 0)
เครือข่าย: hidden 2 โหนด (ReLU): h₁: w=(1,1), b=−0.5 · h₂: w=(1,1), b=−1.5 · output: w=(2,−8), b=0 → sigmoid ขั้น 1 — ชั้นซ่อน: z₁ = 1·1 + 1·0 − 0.5 = 0.5 → h₁ = ReLU(0.5) = 0.5 z₂ = 1·1 + 1·0 − 1.5 = −0.5 → h₂ = ReLU(−0.5) = 0 ขั้น 2 — ชั้นออก: z = 2·(0.5) + (−8)·0 + 0 = 1.0 ขั้น 3 — sigmoid: σ(1.0) = 1/(1+e⁻¹) = 0.73 ความน่าจะเป็น 0.73 → ทำนาย คลาส 1 — ตรงกับ XOR(1,0) = 1 พอดี
ตัวอย่าง B — input (1, 1): ทำไมโหนด h₂ ถึงมีไว้ "หักล้าง"
ขั้น 1 — ชั้นซ่อน: z₁ = 1·1 + 1·1 − 0.5 = 1.5 → h₁ = 1.5 z₂ = 1·1 + 1·1 − 1.5 = 0.5 → h₂ = 0.5  (คราวนี้ h₂ ตื่นแล้ว!) ขั้น 2 — ชั้นออก: z = 2·(1.5) + (−8)·(0.5) = 3 − 4 = −1.0 ขั้น 3 — sigmoid: σ(−1.0) = 0.27 ความน่าจะเป็นต่ำ → ทำนาย คลาส 0 = XOR(1,1) ✓ — h₂ คือ "ยามเฝ้ากรณี (1,1)" ที่โผล่มาหักคะแนนด้วย weight −8 — การแบ่งงานกันของโหนดแบบนี้แหละคือสิ่งที่เครือข่ายเรียนรู้
ตัวอย่าง C — softmax: จากคะแนนดิบ (2, 0) เป็นความน่าจะเป็น
สูตร: p_i = e^{z_i} / (e^{z_1} + e^{z_2}) แทนค่า: e² = 7.39, e⁰ = 1.00 → ผลรวม = 8.39 p₁ = 7.39/8.39 = 0.88 · p₂ = 1.00/8.39 = 0.12 รวมกันได้ 1 พอดี — LLM ทำแบบเดียวกันนี้กับคำศัพท์ ~50,000 คำ แล้ว "สุ่มเลือก" คำถัดไปตามความน่าจะเป็นเหล่านี้

ส่วนที่ 5 — แบบฝึกหัด แบบฝึกหัด

ข้อ 1 (ทำในคาบ) — forward pass ด้วยมือ

ใช้เครือข่ายจากตัวอย่าง A (h₁: w=(1,1), b=−0.5 · h₂: w=(1,1), b=−1.5 · output: w=(2,−8), b=0, sigmoid) จงคำนวณผลของ input (0, 1) และ (0, 0) ทีละขั้น แล้วสรุปว่าตรงกับตาราง XOR หรือไม่

ดูเฉลย (0,1): z₁ = 0+1−0.5 = 0.5 → h₁=0.5 · z₂ = 0+1−1.5 = −0.5 → h₂=0
z = 2(0.5) − 8(0) = 1.0 → σ(1.0) = 0.73 → คลาส 1 = XOR(0,1) ✓
(0,0): z₁ = −0.5 → h₁=0 · z₂ = −1.5 → h₂=0 → z = 0 → σ(0) = 0.50 → ก้ำกึ่งพอดี (ธรรมเนียมใช้ > 0.5 จึงได้คลาส 0) = XOR(0,0) ✓ — สังเกตว่าเคสนี้ "มั่นใจน้อยสุด" เพราะทุกโหนดหลับหมด
ข้อ 2 (ทำในคาบ) — softmax

ชั้นออกให้คะแนนดิบ 3 คลาส: z = (1, 3, 0) จงคำนวณ softmax ทั้งสามค่า (ใช้ e¹=2.72, e³=20.09, e⁰=1) แล้วตอบว่าโมเดล "มั่นใจ" คลาสไหนกี่เปอร์เซ็นต์

ดูเฉลย ผลรวม = 2.72 + 20.09 + 1.00 = 23.81
p₁ = 2.72/23.81 = 0.11 · p₂ = 20.09/23.81 = 0.84 · p₃ = 1.00/23.81 = 0.04 (รวม ≈ 1 ✓)
โมเดลมั่นใจคลาส 2 ประมาณ 84% — สังเกตว่าคะแนนดิบห่างกันแค่ 2 แต้ม แต่ความน่าจะเป็นห่างกัน ~8 เท่า เพราะ e^x ขยายความต่างแบบทวีคูณ
ข้อ 3 (การบ้าน) — ออกแบบนิวรอนเป็นประตูตรรกะ

จงหาค่า w₁, w₂, b ของ perceptron 1 ตัว (ใช้ step function: ยิงเมื่อ z ≥ 0) ที่ทำหน้าที่เป็นประตู AND และอีกชุดที่เป็นประตู OR (ตรวจคำตอบกับ input ทั้ง 4 แบบ) แล้วอธิบายว่าทำไมวิธีเดียวกันนี้ใช้กับ XOR ไม่ได้ (การบ้าน — ไม่มีเฉลย)

ข้อ 4 (การบ้าน) — นับพารามิเตอร์

เครือข่ายอ่านลายมือ MNIST: input 784 → hidden 128 (ReLU) → output 10 (softmax) (ก) มี weight และ bias รวมกี่ตัว (แสดงวิธีนับ) (ข) ถ้าเพิ่ม hidden อีกชั้นขนาด 64 ระหว่าง 128 กับ 10 จะเพิ่มอีกกี่ตัว (ค) จากเวที playground: ทำไมเครือข่ายที่ weight "สุ่ม" จึงตอบมั่ว และเราต้องทำอะไรกับมัน (เขียนเชื่อมไปเรื่อง Backpropagation) (การบ้าน — ไม่มีเฉลย)

ส่วนที่ 6 — ขั้นบันไดของสัปดาห์นี้ ขั้นบันได

สัปดาห์นี้พาเราเข้าใกล้ LLM อย่างไร:
เราเพิ่งสร้าง "หน่วยก่อสร้าง" ตัวจริงของ AI ยุคปัจจุบันด้วยมือ — ข้างใน Transformer ของ GPT/Claude คือ MLP แบบที่เราคำนวณวันนี้ ซ้อนกันหลายสิบชั้น สลับกับกลไก attention (เรื่อง Attention & Transformer) ไม่มีอะไรลึกลับกว่านี้
และ softmax ที่เราคำนวณมือ ก็คือจังหวะสุดท้ายของ LLM ทุกครั้งที่มันพิมพ์: ให้คะแนนคำศัพท์ ~50,000 คำ → softmax → สุ่มเลือก "คำถัดไป" — เกมเดียวกับ bigram เรื่อง AI คืออะไร แต่เครื่องให้คะแนนฉลาดกว่ามหาศาล
ปมที่ค้างไว้: weight ชุด XOR วันนี้เราตั้งให้เอง แต่ GPT มี weight เป็นล้านล้านตัว ใครจะไปตั้งมือไหว? — สัปดาห์หน้า: อัลกอริทึมที่ปลุกวงการจากความตาย: Backpropagation — วิธีให้เครือข่าย "เรียนรู้ weight เอง" ที่เราจะไล่เลขทุกตัวด้วยมือ
