KNN: จำแนกด้วย "เพื่อนบ้านใกล้ที่สุด"

อยากรู้ว่าจุดใหม่เป็นพวกไหน? ดูว่าเพื่อนบ้านที่อยู่ใกล้มันที่สุด k คนเป็นพวกไหน แล้วโหวต — เรียบง่ายจนน่าตกใจ แต่คือรากของ "ค้นด้วยความหมาย" ใน AI ยุคนี้

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายหลักการ K-Nearest Neighbors (KNN) ได้ว่า "จุดใหม่ถูกจัดประเภทตามเสียงข้างมากของเพื่อนบ้าน k จุดที่ใกล้ที่สุด"
  2. คำนวณ ระยะทาง (Euclidean / Manhattan) เรียงลำดับ เลือก k จุด แล้วโหวตด้วยมือได้
  3. อธิบายผลของค่า k ต่อเส้นแบ่งเขต (k เล็ก = ดิ้นตามสัญญาณรบกวน/overfit, k ใหญ่ = เรียบเกิน/underfit) และเลือก k ที่พอดี
  4. เชื่อมโยงได้ว่า KNN บนพิกัด "ความหมาย" (embedding) คือหัวใจของ semantic search / RAG ที่ LLM ใช้ค้นความรู้

ส่วนที่ 1 — แนวคิด: "บอกได้ว่าเป็นใคร จากเพื่อนที่คบ" แนวคิด

คำถามเปิดคาบ: "มีผลไม้ลูกหนึ่งวางอยู่ตรงหน้า เรารู้แค่ น้ำหนัก กับ ความหวาน ของมัน — จะเดายังไงว่ามันคือส้มหรือแอปเปิล?"

สัญชาตญาณของมนุษย์ตอบทันทีว่า: "ดูสิว่ามันเหมือนของที่เรารู้จักอันไหนมากที่สุด" — ถ้ามันคล้ายส้มหลายลูกที่เคยเห็น ก็เดาว่าส้ม นี่แหละคือ KNN เป๊ะๆ ไม่มีการฝึกสมการ ไม่มีการหาน้ำหนัก w เหมือน week02 — อัลกอริทึมแค่ "จำข้อมูลทั้งหมดไว้" แล้วตอนต้องทำนายค่อยไปถามเพื่อนบ้าน

สูตรคิดของ KNN มีแค่ 3 ขั้น (จำง่ายมาก):

① วัดระยะ จากจุดใหม่ ไปยังทุกจุดที่เรามี  →  ② เลือก k จุดที่ใกล้สุด  →  ③ โหวต คลาสไหนมีเยอะสุดในกลุ่มนั้น = คำตอบ

ต่างจาก week02–03 เดิมยังไง? Linear/Polynomial Regression สร้างเส้น/สมการขึ้นมาอธิบายข้อมูล (เรียนรู้ตอนเทรน) — ส่วน KNN ไม่สร้างอะไรเลย มันเลื่อนงานหนักไปไว้ตอนทำนาย จึงถูกเรียกว่า lazy learner (ผู้เรียนขี้เกียจ) และเป็น ตัวจำแนก (classifier) ไม่ใช่ตัวถดถอย — ประตูสู่โลกของ "ระยะทาง" ที่สัปดาห์หน้าเราจะต่อยอดไปสู่ K-means clustering — ให้เครื่องจับกลุ่มข้อมูลเองโดยไม่มีเฉลย (แล้วไปถึง Neural Network ในเวลาต่อมา)

ส่วนที่ 2 — เวทีเล่น: ขยับจุด ปรับ k แล้วดูเส้นแบ่งเขตมีชีวิต ทดลองเล่น

KNN Playground

พื้นหลังสีอ่อน = เขตการตัดสินใจ (decision boundary) ที่ KNN คิดไว้ให้ทุกจุดในสนามล่วงหน้า · แกน X–Y มีตัวเลขกำกับ (0–10) อ่านพิกัดจุดได้ · เลื่อนเมาส์ (หรือแตะบนมือถือ) เพื่อวางจุดปริศนา แล้วดูมันลากเส้นไปหา k เพื่อนบ้านใกล้ที่สุด พร้อมนับโหวตแบบสดๆ · อยากสร้างข้อมูลเอง? กด โหมดวางจุดเอง แล้วเลือกสีคลาส คลิกลงสนามได้เลย

ส่วนที่ 3 — KNN ทำงานทีละขั้น (คำนวณด้วยมือ) ทีละขั้น

โจทย์: มีจุดปริศนา Q = (5, 5) และข้อมูล 6 จุด ( คลาสน้ำเงิน 3 จุด, คลาสแดง 3 จุด) ใช้ k = 3 และระยะแบบ Euclidean — กดปุ่ม เล่นทีละขั้น หรือคลิกที่ขั้นใดก็ได้

  1. ตั้งโจทย์: วางจุดปริศนา Q ลงในสนาม — เรายังไม่รู้ว่ามันเป็นสีอะไร รู้แค่พิกัด (5, 5)
  2. วัดระยะไปทุกจุด: ใช้ d = √((Δx)² + (Δy)²) คำนวณระยะจาก Q ไปยังข้อมูลทั้ง 6 จุด
  3. เรียงจากใกล้ → ไกล: จัดอันดับระยะทางที่ได้ จากน้อยไปมาก
  4. เลือก k = 3 จุดที่ใกล้ที่สุด: ตัดมาเฉพาะ 3 อันดับแรก วงกลมล้อมไว้
  5. โหวตตามเสียงข้างมาก: นับว่าในกลุ่ม 3 จุดนั้น เป็นสีอะไรกี่จุด
  6. สรุปคำตอบ: คลาสที่ได้เสียงมากที่สุดคือคำตอบของ Q

ส่วนที่ 4 — ตัวอย่างคำนวณด้วยมือ (วิธีเดียวกับส่วนที่ 3) ลงมือ

ตัวอย่าง A — จุดเดียวกัน แต่เปลี่ยน k = 5 คำตอบเปลี่ยนไหม?
ขั้น 1 — ระยะทั้ง 6 จุดจาก Q(5,5): น้ำเงิน A₂(6,5)=1.00, A₁(4,6)=1.41, A₃(7,7)=2.83 · แดง B₁(2,2)=4.24, B₂(3,1)=4.47, B₃(1,3)=4.47 ขั้น 2 — เรียงใกล้→ไกล แล้วหยิบ 5 อันดับแรก (k=5): A₂(1.00), A₁(1.41), A₃(2.83), B₁(4.24), B₂(4.47) — ได้ 3 จุด, 2 จุด ขั้น 3 — โหวต: 3 เสียง มากกว่า 2 เสียง คำตอบยังเป็น น้ำเงิน — แต่คราวนี้ "เฉียด" กว่าเดิม (3:2) แสดงว่า Q อยู่ค่อนไปทางรอยต่อ ถ้า k โตกว่านี้อีกอาจพลิกได้
ตัวอย่าง B — เปลี่ยนเป็นระยะ Manhattan (L1) ที่ k = 3
ขั้น 1 — Manhattan d = |Δx| + |Δy| จาก Q(5,5): A₂(6,5)=1, A₁(4,6)=2, A₃(7,7)=4 · B₁(2,2)=6, B₂(3,1)=6, B₃(1,3)=6 ขั้น 2 — 3 อันดับแรก: A₂(1), A₁(2), A₃(4) ล้วนเป็น คำตอบ น้ำเงิน (3:0) — เมตริกต่างกันได้ระยะต่างกัน แต่ในกรณีนี้ "เพื่อนบ้าน 3 อันดับแรก" ยังเป็นชุดเดิม คำตอบเลยไม่เปลี่ยน (บางโจทย์เปลี่ยนเมตริกแล้วเพื่อนบ้านสลับได้!)
ตัวอย่าง C — KNN ทำ "ถดถอย" (regression) ก็ได้ ไม่ใช่แค่จำแนก
โจทย์: ทำนาย ราคาบ้าน ของหลังใหม่ จากบ้าน k=3 หลังที่คล้ายที่สุด ซึ่งมีราคา 3.0, 3.4, 3.2 ล้าน ขั้น 1 — KNN Regression ไม่โหวต แต่ "เฉลี่ย": ŷ = (3.0 + 3.4 + 3.2) / 3 ได้ 3.2 ล้าน — เปลี่ยนจาก "โหวตหาคลาส" เป็น "เฉลี่ยค่าตัวเลข" เท่านี้ KNN ก็ทำนายค่าต่อเนื่องได้ (ถ้าถ่วงน้ำหนัก ก็ใช้ค่าเฉลี่ยถ่วงน้ำหนักตามระยะ)

ส่วนที่ 5 — แบบฝึกหัด แบบฝึกหัด

ข้อ 1 (ทำในคาบ) — โหวตด้วยมือ

จุดปริศนา Q = (0, 0) มีข้อมูล 5 จุด:
G₁(1,1), G₂(2,0), G₃(0,2)  ·  R₁(3,1), R₂(1,3)
ใช้ระยะ Euclidean และ k = 3 จงหาว่า Q เป็นคลาสอะไร (แสดงระยะทุกจุด, เรียงลำดับ, แล้วโหวต)

ดูเฉลย ระยะจาก Q(0,0): G₁=√2≈1.41, G₂=2.00, G₃=2.00, R₁=√10≈3.16, R₂=√10≈3.16
เรียงใกล้สุด 3 อันดับ: G₁(1.41), G₂(2.00), G₃(2.00) → ทั้งหมด
โหวต 3 : 0 → คำตอบ = เขียว
ข้อ 2 (ทำในคาบ) — k สำคัญแค่ไหน

ใช้ข้อมูลชุดเดิมกับข้อ 1 แต่ย้ายจุดปริศนาเป็น Q = (2, 2)
(ก) ที่ k = 1 Q เป็นคลาสอะไร?   (ข) ที่ k = 5 (ใช้ทุกจุด) Q เป็นคลาสอะไร?   (ค) อธิบายว่าทำไมคำตอบต่างกัน

ดูเฉลย ระยะจาก Q(2,2): G₁(1,1)=√2≈1.41, G₂(2,0)=2.00, G₃(0,2)=2.00, R₁(3,1)=√2≈1.41, R₂(1,3)=√2≈1.41
(ก) k=1: จุดที่ใกล้สุดเสมอกัน 3 จุด (G₁, R₁, R₂ ที่ระยะ 1.41) — ถ้า tie-break เลือกจุดแรก/สุ่ม อาจได้ หรือ → คำตอบไม่แน่นอน (บทเรียน: k=1 เปราะต่อ noise และ tie มาก)
(ข) k=5: ใช้ทุกจุด → 3 : 2 → เขียว
(ค) k เล็กมองแค่จุดใกล้สุดจึงไวและไม่เสถียร ส่วน k ใหญ่มองภาพรวมของบริเวณ จึงให้คำตอบที่ "เห็นพ้องของหมู่มาก" มั่นคงกว่า
ข้อ 3 (ทำในคาบ) — ทำไมต้องปรับสเกล

ชุดข้อมูลมีฟีเจอร์ "เงินเดือน (บาท)" ช่วง 15000–80000 กับ "อายุ (ปี)" ช่วง 22–60 ถ้าใช้ KNN โดย ไม่ ปรับสเกล จงอธิบายว่าฟีเจอร์ใดจะครอบงำการวัดระยะ และควรแก้อย่างไร

ดูเฉลย "เงินเดือน" มีช่วงกว้างเป็นหมื่น ส่วน "อายุ" ต่างกันแค่หลักสิบ — ในสูตรระยะ ผลต่างเงินเดือนยกกำลังสองจะมหาศาลจนกลบผลต่างอายุไปเกือบหมด ทำให้ KNN ตัดสินโดยดู "เงินเดือน" อย่างเดียว
แก้ด้วย feature scaling: standardization x←(x−μ)/σ หรือ min–max scaling ให้ทุกฟีเจอร์อยู่สเกล 0–1 เท่ากันก่อนคำนวณระยะ
ข้อ 4 (การบ้าน) — เลือก k ด้วยเหตุผล

ชุดข้อมูลมี 100 จุด แบ่ง 2 คลาส มี noise พอสมควร (ก) เสนอค่า k เริ่มต้นที่สมเหตุสมผล พร้อมเหตุผล (คำนวณจาก √n ประกอบ) (ข) ทำไมนิยมเลือก k เป็นเลขคี่ในปัญหา 2 คลาส (ค) อธิบายวิธี cross-validation สั้นๆ ว่าใช้เลือก k ที่ดีที่สุดได้อย่างไร (การบ้าน — ไม่มีเฉลย)

ข้อ 5 (การบ้าน) — โยงสู่ AI จริง

ระบบแนะนำเพลง "คนที่ชอบเพลงคล้ายคุณ ก็ชอบเพลงนี้" และระบบ RAG ที่ LLM "ค้นเอกสารที่เกี่ยวข้องกับคำถาม" ทั้งสองใช้แนวคิด nearest-neighbor เหมือนกัน จงอธิบายว่า "จุด" และ "ระยะทาง" ในสองระบบนี้คืออะไร และเหตุใดการมี embedding ที่ดี จึงสำคัญกว่าตัวอัลกอริทึม KNN เอง (การบ้าน — ไม่มีเฉลย)

ส่วนที่ 6 — ขั้นบันไดของสัปดาห์นี้ ขั้นบันได

สัปดาห์นี้พาเราเข้าใกล้ LLM อย่างไร:
KNN สอนบทเรียนที่ลึกที่สุดข้อหนึ่งของ AI: "ความหมาย = ความใกล้ในปริภูมิ" — สิ่งที่คล้ายกันควรอยู่ใกล้กัน แล้วเราตัดสินของใหม่จากเพื่อนบ้านของมัน
เมื่อ LLM แปลงคำ/ประโยค/เอกสารเป็นเวกเตอร์ embedding (พิกัดในปริภูมิความหมายหลายร้อยมิติ) การ "ค้นความรู้ที่เกี่ยวข้อง" ก็คือ KNN บนเวกเตอร์เหล่านั้น — นี่คือหัวใจของ semantic search และ RAG (Retrieval-Augmented Generation) ที่ทำให้ ChatGPT ตอบจากเอกสารของเราได้ · vector database (FAISS, Pinecone, HNSW) ที่คุณจะได้ยินบ่อยๆ ก็คือ "KNN เวอร์ชันเร็วสุดขีด" นั่นเอง
สัปดาห์หน้า: เราจะใช้แนวคิด "ระยะทาง" เดิม แต่พลิกเป็นโจทย์ไร้เฉลย — K-means clustering ที่ให้เครื่อง จับกลุ่ม ข้อมูลเอง (unsupervised) — แล้วจึงเข้าสู่ Neural Network และในที่สุด เรียนรู้วิธีสร้าง embedding ที่ทำให้ระยะทางมีความหมายจริงๆ
