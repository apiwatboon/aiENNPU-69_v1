👣 ทบทวนทีละขั้น — แผน 6 ขั้นก่อนสอบ (เรียงตามนี้ได้ผลกว่าอ่านวนๆ)

  1. ไล่ตารางแก่นข้างบน ทีละแถว แถวไหนอ่านแล้ว "เล่าต่อให้เพื่อนฟังได้" ติ๊กผ่าน แถวไหนตะกุกตะกัก จดไว้ — นี่คือรายการที่ต้องกลับไปเก็บ
  2. กลับไปอ่านเฉพาะ "เข้าใจทีละขั้น" ของสัปดาห์ที่ติ๊กไม่ผ่าน ทุกสัปดาห์มีส่วน 👣 ที่ไล่กลไกด้วยตัวเลขจริง — อ่านพร้อมคำนวณตามด้วยมือ เร็วกว่าอ่านเนื้อหาเต็มทั้งหน้า
  3. ทำแบบฝึกหัด "ทำในคาบ" ของทุกเรื่องก่อนสอบใหม่ โดยไม่ดูเฉลย ข้อสอบออกแนวเดียวกันทุกข้อ — ทำได้โดยไม่แอบดู = พร้อมจริง
  4. ข้อที่พลาด: เล่น demo ของสัปดาห์นั้นอีกรอบ ความเข้าใจจากการ "เห็นมันขยับ" ติดทนกว่าการอ่านซ้ำ — โดยเฉพาะ gradient descent (ส.5) และ overfitting (ส.6)
  5. ทำข้อสอบตัวอย่าง 3 ข้อข้างล่าง จับเวลา ข้อละไม่เกิน 10 นาที ทำเสร็จค่อยเปิดเฉลย — ฝึก "เชื่อมข้ามสัปดาห์" เพราะข้อสอบจริงชอบถามแบบนั้น
  6. ปิดท้าย: เล่าเส้นเรื่องทั้งเทอมใน 3 นาที จาก bigram (นับ) → regression (หมุนปุ่ม) → classification (ความน่าจะเป็น) → ซ้อนชั้น → backprop → overfit → CNN — เล่าให้เพื่อน/กระจก/แมวฟังก็ได้ ถ้าเล่าลื่นโดยไม่เปิดโน้ต แปลว่าโครงความรู้ครบแล้ว

วอร์ม 1 — softmax กับคะแนนเสมอกัน

คะแนนดิบ 3 คำเท่ากันหมด: 1, 1, 1 — ความน่าจะเป็นแต่ละคำเท่าไหร่? แล้วถ้าเปลี่ยน temperature ผลเปลี่ยนไหม?

ขั้น 1: ยกกำลัง e — ได้ e¹ เท่ากันทั้งสามตัว
ขั้น 2: หารด้วยผลรวม — เท่ากันสามตัวย่อมได้อย่างละ 1/3 ≈ 33%
ขั้น 3: ลองใส่ T — (1,1,1)÷T ก็ยังเท่ากันหมด ความต่างเป็นศูนย์เหมือนเดิม → ผลไม่เปลี่ยน
✅ คำตอบ: 33% เท่ากันทุกคำ ไม่ว่า T เท่าไหร่ — softmax สนใจแค่ "ความต่าง" ของคะแนน
วอร์ม 2 — ทำไมต้องมี activation function

ถ้าตัด activation ออกจากเครือข่าย 100 ชั้น จะเกิดอะไรขึ้น? (ตอบสั้นๆ 1 ประโยค)

ขั้น 1: ชั้นไม่มี activation = การคูณ weight แล้วบวก = ฟังก์ชันเชิงเส้น
ขั้น 2: เชิงเส้นซ้อนเชิงเส้น (w₂(w₁x + b₁) + b₂) กระจายออกมา = เชิงเส้นเส้นเดียว (w'x + b')
ขั้น 3: ทำซ้ำ 100 ชั้นก็ยุบเหลือเส้นเดียวอยู่ดี
✅ คำตอบ: 100 ชั้นมีพลังเท่าชั้นเดียว แก้ XOR ไม่ได้เหมือนเดิม — activation คือ "ข้อหักงอ" ที่ทำให้ความลึกมีความหมาย
วอร์ม 3 — gradient descent ที่จบใน 1 ก้าว

L(w) = (w−2)² เริ่ม w = 5, learning rate = 0.5 — คำนวณ 2 ก้าว แล้วอธิบายสิ่งที่เห็น

ขั้น 1 — ก้าวแรก: ชันที่ w=5: 2(5−2) = 6 → w = 5 − 0.5(6) = 2 — ถึงคำตอบเป๊ะ!
ขั้น 2 — ก้าวสอง: ชันที่ w=2: 2(2−2) = 0 → ไม่ขยับอีก (ก้นหุบ ความชันศูนย์)
ขั้น 3 — ตีความ: lr = 0.5 บนพาราโบลา (w−c)² คือก้าวพอดีเผง: กระโดดถึงจุดต่ำสุดเสมอไม่ว่าเริ่มที่ไหน
✅ คำตอบ: ถึง w = 2 ใน 1 ก้าวแล้วหยุดนิ่ง — แต่อย่าหลงดีใจ: loss จริงไม่ใช่พาราโบลาสวยๆ จึงไม่มี lr วิเศษแบบนี้ในงานจริง
วอร์ม 4 — bigram ความเร็วสูง

ข้อมูลฝึก: "ABAB" — หลังตัว A โมเดลทำนายอะไร ด้วยความน่าจะเป็นเท่าไหร่?

ขั้น 1: จับคู่จาก A-B-A-B: AB, BA, AB
ขั้น 2: นับหลัง A: B 2 ครั้ง จาก 2 ครั้ง
✅ คำตอบ: B 100% (หลัง B ก็ A 100% เช่นกัน) — โมเดลนี้แต่งข้อความได้แบบเดียว: ABABAB...
วอร์ม 5 — MSE สองโมเดลใน 30 วินาที

ข้อมูล (1,2), (2,4) — โมเดล ก: ŷ = 2x | โมเดล ข: ŷ = x+1 — ใคร MSE ต่ำกว่า?

ขั้น 1 — โมเดล ก ทำนาย: 2(1)=2 ✓, 2(2)=4 ✓ → ผิด 0, 0 → MSE = 0
ขั้น 2 — โมเดล ข ทำนาย: 1+1=2 ✓, 2+1=3 (จริง 4 ผิด 1) → MSE = (0+1)/2 = 0.5
✅ คำตอบ: ก ชนะ (MSE 0 ต่อ 0.5)
วอร์ม 6 — sigmoid ที่ศูนย์

sig(0) เท่ากับเท่าไหร่ และทำไมค่านี้ถึง "ต้อง" เป็นแบบนั้น?

ขั้น 1 — แทนค่าในสูตร: sig(0) = 1/(1+e⁻⁰) = 1/(1+1) = 1/2
ขั้น 2 — ตีความ: คะแนนดิบศูนย์ = หลักฐานสองฝั่งสมดุล ไม่เอียงข้างไหน → ความน่าจะเป็นต้องกลางพอดี
✅ คำตอบ: 50% เสมอ — จุด z = 0 คือ decision boundary โดยนิยาม
วอร์ม 7 — XOR หนึ่งประโยค

อธิบายใน 1 ประโยคว่าทำไม perceptron ชั้นเดียวทำ XOR ไม่ได้

ขั้น 1 — แปลง perceptron เป็นภาพ: ชั้นเดียว = เส้นตรงแบ่งระนาบหนึ่งเส้น
ขั้น 2 — วาดจุด XOR: กลุ่ม "1" อยู่มุม (1,0) กับ (0,1) | กลุ่ม "0" อยู่มุม (0,0) กับ (1,1) — สลับทแยงกัน
ขั้น 3 — ลองลากเส้น: เส้นตรงเส้นเดียวแยกคู่ทแยงไม่ได้ (ลองดูในกระดาษ — แยกได้อย่างมาก 3 จาก 4 จุด)
✅ คำตอบ: เพราะ XOR ไม่ใช่ปัญหาที่เส้นตรงเส้นเดียวแบ่งได้ — ต้องซ้อนชั้นเพื่อสร้างเส้นแบ่งหักงอ
วอร์ม 8 — กับดักการวินิจฉัย: train 0.05 / test 0.07

train loss ต่ำมากขนาดนี้ overfit หรือเปล่า?

ขั้น 1 — ทวนนิยาม overfit: ตัดสินจากช่องว่างระหว่าง train กับ test ไม่ใช่ค่า train เดี่ยวๆ
ขั้น 2 — วัดช่องว่าง: 0.07 − 0.05 = 0.02 — แคบมาก
ขั้น 3 — สรุปอาการ: ต่ำทั้งคู่ + ช่องว่างแคบ = เรียนรู้ได้ดีและ generalize ได้ดี
✅ คำตอบ: ไม่ overfit — นี่คือโมเดลในฝัน ข้อสอบชอบหลอกด้วย "train ต่ำ = overfit" ซึ่งผิด
วอร์ม 9 — kernel ที่ไม่ทำอะไรเลย

kernel 3×3 ที่เป็นศูนย์ทุกช่อง ยกเว้นช่องกลาง = 1 — ภาพที่ผ่าน kernel นี้หน้าตาเป็นยังไง?

ขั้น 1 — ทาบ kernel ที่ตำแหน่งใดก็ได้: คูณช่องต่อช่อง — รอบข้าง 8 ช่อง × 0 = 0 หมด เหลือแค่ช่องกลาง × 1
ขั้น 2 — บวกรวม: output = ค่า pixel กลางเป๊ะๆ
ขั้น 3 — ทำทุกตำแหน่ง: ทุก output = pixel เดิมของมัน
✅ คำตอบ: ได้ภาพเดิมเป๊ะ (identity kernel) — ถ้าอธิบายข้อนี้ได้ แปลว่าเข้าใจกลไก convolution จริง
วอร์ม 10 — จริงหรือเท็จ: "GPT เทรนด้วย MSE"

ตอบพร้อมเหตุผลสั้นๆ — ระวังข้อแบบนี้ในข้อสอบ

ขั้น 1 — ถามก่อน: GPT ทำนายอะไร? เลือก token ถัดไปจากคลังศัพท์ = โจทย์หมวดหมู่ (classification) ไม่ใช่ตัวเลขต่อเนื่อง
ขั้น 2 — จับคู่ loss ให้ตรงชนิดโจทย์: หมวดหมู่ → cross-entropy (−log P ของคำตอบจริง) | ตัวเลข → MSE
✅ คำตอบ: เท็จ — GPT ใช้ cross-entropy (ส่วน diffusion ที่ทำนาย noise เป็นตัวเลข ใช้ MSE — ระวังข้อหลอกแบบนี้)

📝 ตัวอย่างข้อ 1 — เชื่อมโยงข้ามสัปดาห์

"การเทรน GPT ใช้แนวคิดจากทุกสัปดาห์ที่เรียนมา" — จงยกแนวคิดจาก 3 สัปดาห์ใดก็ได้ แล้วอธิบายว่าแต่ละแนวคิดไปโผล่ตรงไหนในการเทรน LLM (6 คะแนน)

แนวคำตอบ ตัวอย่าง: (ส.2) weight/loss — LLM มี weight แสนล้านตัว loss คือทำนายคำถัดไปผิดแค่ไหน | (ส.3) softmax — ทุก token ที่พิมพ์คือ softmax เหนือคำศัพท์ทั้งหมด | (ส.5) backprop — วิธีอัปเดต weight ทั้งหมดคือ backprop วนหลายล้านรอบ | (ส.6) ข้อมูลมหาศาล — กัน overfit ฯลฯ (เลือก 3 จากนี้หรือเทียบเท่า)
📝 ตัวอย่างข้อ 2 — คำนวณผสม

โมเดล ŷ = wx โดย w = 2 ข้อมูล (x, y) = (3, 9) และ loss = (ŷ − y)², learning rate = 0.05
(ก) หา loss ปัจจุบัน (ข) หา gradient dL/dw (ค) อัปเดต w หนึ่งก้าว (ง) loss ใหม่ลดลงจริงไหม

ดูเฉลย (ก) ŷ = 6, loss = 9   (ข) dL/dw = 2(ŷ−y)·x = 2(−3)(3) = −18   (ค) w = 2 − 0.05(−18) = 2.9   (ง) ŷ = 8.7, loss = 0.09 — ลดจาก 9 เหลือ 0.09 ✓
📝 ตัวอย่างข้อ 3 — วิเคราะห์สถานการณ์

สตาร์ทอัพเทรนโมเดลตรวจมะเร็งจากภาพ X-ray ของโรงพยาบาล A ได้ accuracy 97% บน test set (ที่มาจากโรงพยาบาล A เช่นกัน) แต่พอใช้จริงที่โรงพยาบาล B ได้แค่ 62% — จงอธิบายปรากฏการณ์นี้ด้วยแนวคิดที่เรียนมา และเสนอวิธีแก้ 2 วิธี

แนวคำตอบ โมเดล overfit กับลักษณะเฉพาะของโรงพยาบาล A (เครื่องถ่าย, มุม, ประชากรผู้ป่วย) — train และ test มาจากแหล่งเดียวกันจึงวัด "ความเข้าใจโรค" ไม่ได้จริง (distribution shift)
วิธีแก้: ผสมข้อมูลหลายโรงพยาบาลตอนเทรน / ทำ test set ข้ามโรงพยาบาล / data augmentation / fine-tune กับข้อมูลโรงพยาบาล B

กิจกรรมทบทวนในคาบ (ก่อนสอบ)

เกม "ต่อบันไดให้ครบ": แบ่งกลุ่ม กลุ่มละ 4–5 คน แจกการ์ดคำศัพท์ครึ่งแรกของเทอม (weight, loss, gradient, softmax, activation, overfit, kernel, feature map, ...) ให้แต่ละกลุ่มเรียงการ์ดเป็น "เส้นเรื่องจาก bigram → CNN" แล้วเล่าหน้าห้อง 3 นาที ว่าแนวคิดไหนเกิดมาเพื่อแก้ปัญหาของแนวคิดก่อนหน้า — กลุ่มที่เล่าเชื่อมโยงได้ลื่นที่สุดได้คะแนนพิเศษ

ส่วนที่ 4 — ขั้นบันได ขั้นบันได

🪜 มาถึงครึ่งทาง: เรามี "เครื่องทำนายที่เรียนรู้ได้" ครบเครื่อง — โมเดล, loss, วิธีเทรน, วิธีวัดผล และบทเรียนเรื่องสถาปัตยกรรม
ครึ่งหลังของเทอม: เอาเครื่องนี้มา "เข้าใจภาษา" — เริ่มจากความพยายามแรก (RNN) ที่ล้มเหลวอย่างมีบทเรียน ไปจนถึง Transformer และ LLM ตัวจริง
