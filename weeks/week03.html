Polynomial Regression (การถดถอยพหุนาม)

เมื่อข้อมูลไม่เป็นเส้นตรง — เพิ่ม x², x³ แล้ว "ดัดเส้นให้โค้ง" เข้ากับข้อมูล (และเรียนรู้กับดัก overfitting)

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายได้ว่า Polynomial Regression คือ Linear Regression ที่ "เพิ่มฟีเจอร์ x², x³, …" และ degree (อันดับของพหุนาม) คืออะไร
  2. ตั้ง normal equations จากตารางผลรวมกำลัง แล้วแก้หา a₀, a₁, …, aₙ เพื่อได้สมการเส้นโค้ง y = a₀ + a₁x + a₂x² + …
  3. อธิบาย underfit / overfit และเลือก degree ที่ "พอดี" กับข้อมูลได้

ส่วนที่ 1 — แนวคิด: จากเส้นตรงสู่เส้นโค้ง แนวคิด

บางทีข้อมูลจริง ไม่ได้เรียงเป็นเส้นตรง — เช่น ระยะเบรกที่เพิ่มแบบกำลังสองตามความเร็ว หรือทักษะที่ "อิ่มตัว" เมื่อซ้อมมากขึ้น ถ้าใช้เส้นตรงลากจะไม่เข้ากับข้อมูล วิธีแก้ที่เรียบง่ายอย่างน่าทึ่งคือ เพิ่มพจน์กำลังสูงขึ้น:

y = a₀ + a₁x + a₂x² + a₃x³ + … + aₙxⁿ

กุญแจสำคัญ: ถึงจะมี x², x³ แต่มัน ยังเป็น "เชิงเส้นในเชิงสัมประสิทธิ์" — เราแค่มองว่า x, x², x³ เป็น "คอลัมน์ฟีเจอร์" ใหม่ แล้วทำ Least Squares เหมือน week02 · เลข degree (อันดับ n) คือกำลังสูงสุด — degree 1 = เส้นตรง, 2 = พาราโบลา, 3 = ตัว S, ยิ่งสูงยิ่งดัดโค้งได้อิสระ

โอ้โหโมเมนต์: มองในสามมิติ — "เส้นโค้ง" ที่แท้คือ "ระนาบแบนๆ" ในมิติที่สูงกว่า

ความลับที่ใหญ่ที่สุดของสัปดาห์นี้: การเพิ่มฟีเจอร์ x² ก็คือการ "ยก" ข้อมูลขึ้นสู่มิติใหม่ — ในโลก 2 มิติจุดพวกนี้ต้องใช้เส้นโค้ง แต่พอเพิ่มแกน z = x² เข้าไป ระนาบแบนๆ (เชิงเส้นแท้ๆ!) กลับฟิตพอดีเป๊ะ — กดเล่นทั้ง 4 ฉาก แล้วลากหมุนดูเองได้เลย

🖱️ ลากเพื่อหมุน
ฉาก 1 เส้นตรงแพ้ · ฉาก 2 ยกมิติ z=x² · ฉาก 3 ระนาบฟิตเป๊ะ · ฉาก 4 มองกลับ = พาราโบลา!

ส่วนที่ 2 — เวทีเล่น: ดัด degree แล้วดู Overfitting เกิดต่อหน้า ทดลองเล่น

🎛️ Polynomial Playground — เห็นทั้ง "เส้นดัดสด" และ "กราฟรูปตัว U วาดตัวเอง"

ข้อมูลถูกแบ่งเป็น ● ชุดฝึก (ใช้ฟิตเส้น) กับ ◆ ชุดทดสอบ (เก็บไว้จับผิด!) — ลาก degree จาก 1 → 9 แล้วดู 2 อย่างพร้อมกัน: เส้นบนกราฟเริ่ม "ดิ้น" เก็บทุกจุด และกราฟขวามือ MSE ทดสอบดีดกลับเป็นรูปตัว U — นาทีที่เข้าใจ overfitting ตลอดชีวิต · ลากจุดได้ กดทัวร์ให้เล่นเองก็ได้

OLS = Ordinary Least Squares (วิธีกำลังสองน้อยสุด) — วิธีมาตรฐานในการหาค่าสัมประสิทธิ์ โดยเลือกค่าที่ทำให้ ผลรวมความคลาดเคลื่อนกำลังสอง (Σ error²) น้อยที่สุด · มีสูตรปิด β = (XᵀX)⁻¹Xᵀy

1. หลักการพื้นฐาน — Linear vs Polynomial

ถ้าเป็น Linear Regression สมการเป็นเส้นตรงเส้นเดียว:

y = β₀ + β₁x

แต่ถ้าความสัมพันธ์เป็น เส้นโค้ง เราใช้ พหุนาม (Polynomial) โดยเพิ่มกำลังของ x:

ดีกรี 2 (Quadratic):   y = β₀ + β₁x + β₂x²
ดีกรี 3 (Cubic):   y = β₀ + β₁x + β₂x² + β₃x³

β₀ = ค่าคงที่ (intercept) · β₁, β₂, β₃, … = สัมประสิทธิ์ (coefficients) · สิ่งที่เราต้องหาคือ β ทุกตัว

2–3. ตั้งรูปสมการกับโจทย์

มองข้อมูลแล้วเป็นเส้นโค้ง → เลือก Polynomial degree = 2 · ตั้งรูปสมการ (ใช้ a, b, c แทน β₀, β₁, β₂ ให้อ่านง่าย):

y = a + bx + cx²

ตอนนี้ ยังไม่รู้ว่า a เท่าไร, b เท่าไร, c เท่าไร — ต้องหาจากข้อมูล

6. ข้อมูลมากกว่าจำนวนตัวแปร → ใช้ Least Squares · ทฤษฎี

Least Squares (วิธีกำลังสองน้อยสุด) คืออะไร? — วิธีเลือกค่าสัมประสิทธิ์ที่ทำให้เส้น "เข้าใกล้ข้อมูลมากที่สุด" โดยวัดความใกล้จาก ผลรวมของ (ความคลาดเคลื่อน)²

เหตุที่ต้องมีวิธีนี้: ในโลกจริงมักมีข้อมูลเยอะ (เช่น 100 จุด) แต่หาแค่ 3 ตัว (a, b, c) → ไม่มีเส้นไหนผ่านทุกจุดพอดี จึงขอ "เส้นที่ดีที่สุด" แทน

error ของแต่ละจุด = ค่าจริง − ค่าที่เส้นทาย: ei = yi − ŷi · แล้วนิยาม "ความไม่เข้ากัน" ของเส้นด้วยผลรวมกำลังสอง:

SSE = Σ (yi − ŷi)²  →  ยิ่งน้อยยิ่งดี

Least Squares = เลือก a, b, c ที่ทำให้ SSE นี้ น้อยที่สุด (least = น้อยสุด · squares = กำลังสองของ error)

ทำไมต้องยกกำลังสอง?

  • กัน error บวก/ลบ หักล้างกัน (ไม่งั้นเส้นแย่ ๆ อาจได้ผลรวมเป็น 0)
  • ลงโทษจุดที่พลาดมาก เป็นพิเศษ (พลาด 2 เท่า → โทษ 4 เท่า)
  • คณิตศาสตร์สวย: หาอนุพันธ์ให้ = 0 ได้ คำตอบเดียวแบบสูตรปิด (คือข้อ 7)

มุมเรขาคณิต: Least Squares คือการ "ฉายเงา (project)" เวกเตอร์ y ลงบนปริภูมิที่เส้นโมเดลไปถึงได้ — จุดที่ใกล้ที่สุดคือจุดที่ error ตั้งฉาก กับปริภูมินั้นพอดี ซึ่งนำไปสู่ normal equations XᵀX·β = Xᵀy โดยตรง

7. สูตรหาสัมประสิทธิ์ (Closed-form Solution) · ทฤษฎี

สูตรนี้มาจากไหน? เชื่อมกับข้อ 6 ยังไง — ข้อ 6 อยากได้ β ที่ทำให้ SSE น้อยสุด · เขียน SSE เป็นเวกเตอร์ (เพราะ ŷ = Xβ): SSE = ‖y − Xβ‖²

หา "จุดต่ำสุด" = ตรงที่อนุพันธ์เทียบ β เป็นศูนย์ แล้วจัดรูป:

①  ∂SSE/∂β = −2Xᵀ(y − Xβ) = 0
②  → XᵀXβ = Xᵀy  (normal equations)
③  → β = (XᵀX)⁻¹Xᵀy  (คูณ (XᵀX)⁻¹ ทั้งสองข้าง)

จึงได้ คำตอบแบบปิด (แก้ได้ตรง ๆ ไม่ต้องเดา):

β = (XᵀX)⁻¹ Xᵀy
💡 คิดง่าย ๆ: (XᵀX)⁻¹ คือ "เวอร์ชันเมทริกซ์ของการหารด้วย XᵀX" — สูตรนี้ก็คือ "Xᵀy หารด้วย XᵀX" นั่นเอง · ถ้ามีตัวแปรเดียว (y = βx) มันคือ β = Σxy / Σx² หารกันตรง ๆ เลย! · พอมีหลายสัมประสิทธิ์ ตัวเลขกลายเป็นเมทริกซ์ที่หารตรง ๆ ไม่ได้ จึงใช้ อินเวอร์ส (⁻¹) แทน (และต้องอยู่ ซ้าย เพราะเมทริกซ์คูณสลับที่ไม่ได้)

ขั้นตอนใช้งาน: สร้าง X → คำนวณ XᵀX → หาผกผัน (inverse) → คูณกับ Xᵀy → ได้ a, b, c (ลงมือจริงในข้อ 8)

8. คำตอบของโจทย์ — ลงมือคำนวณจริง

ขั้น 1 — คำนวณ XᵀX และ Xᵀy จากข้อมูล · XᵀX มาจากเอา Xᵀ คูณ X — ทุกช่องกลายเป็น "ผลรวมกำลังของ x" โดยช่องแถว i คอลัมน์ j = Σxi+j

Xᵀ อ่านว่า "X ทรานสโพส (transpose)" = พลิกเมทริกซ์ สลับแถว ↔ คอลัมน์ · คอลัมน์ของ X (สีเดียวกัน) → ย้ายไปเป็นแถวของ Xᵀ · 🖱️ กดเลขช่องไหนก็ได้ เพื่อไฮไลต์ทั้งคอลัมน์/แถวนั้น:

สีเดียวกัน = คอลัมน์ของ X ที่กลายเป็นแถวของ Xᵀ · ■ ค่า 1 · ■ x · ■ x²

แล้ว Xᵀ (3×5) คูณ X (5×3) = XᵀX (3×3) — แต่ละช่องคือผลรวมของผลคูณ = ผลรวมกำลังของ x

คำนวณผลรวมที่ต้องใช้ก่อน (x = 1,2,3,4,5 · y = 2,5,10,17,26):

n = 5  ·  Σx = 1+2+3+4+5 = 15  ·  Σx² = 1+4+9+16+25 = 55
Σx³ = 1+8+27+64+125 = 225  ·  Σx⁴ = 1+16+81+256+625 = 979
Σy = 60  ·  Σxy = 2+10+30+68+130 = 240  ·  Σx²y = 2+20+90+272+650 = 1034
① XᵀX = Xᵀ × X

🖱️ กดเลขในเมทริกซ์ XᵀX — มันมาจาก แถวของ Xᵀ × คอลัมน์ของ X ช่องไหน และตรงกับเลขไหนในสมการ (XᵀX สมมาตร เลขจึงซ้ำเป็นคู่) · สี: แถว Xᵀ × คอลัมน์ X = ผลลัพธ์

② Xᵀy = Xᵀ × y

🖱️ กดเลขในเวกเตอร์ Xᵀy — มันมาจาก แถวของ Xᵀ × เวกเตอร์ y · สี: แถว Xᵀ × y = ผลลัพธ์

ขั้น 2 — ได้ระบบสมการ (normal equations) XᵀX·β = Xᵀy:

  5a + 15b +  55c =   60
15a + 55b + 225c =  240
55a + 225b + 979c = 1034

ขั้น 3 — แก้ 3 สมการจากขั้น 2 [เรียก (1) (2) (3) ตามลำดับ] · ทำมือด้วย การกำจัดตัวแปรทีละตัว (การหาอินเวอร์ส 3×3 ด้วยมือยุ่งกว่า — ปล่อยให้คอมพิวเตอร์ทำ):

• กำจัด a (ทำให้ a หายไป โดยลบด้วยตัวคูณของสมการ (1)):
  (2) − 3×(1) → 10b + 60c = 60  →  b + 6c = 6  … (2′)
  (3) − 11×(1) → 60b + 374c = 374  … (3′)
• กำจัด b:  (3′) − 60×(2′) → 14c = 14  →  c = 1
• แทนกลับ:  (2′): b + 6(1) = 6 → b = 0  ·  (1): 5a + 15(0) + 55(1) = 60 → 5a = 5 → a = 1
ได้  a = 1  ·  b = 0  ·  c = 1

ตรวจแทนกลับในสมการ: 5(1) + 15(0) + 55(1) = 60 ✓ · 15 + 0 + 225 = 240 ✓ · 55 + 0 + 979 = 1034 ✓ — เข้าครบทั้ง 3 สมการ

🔧 แก้ระบบได้หลายวิธี — ไม่ใช่วิธีเดียว (ทุกวิธีให้คำตอบเดียวกัน ต่างกันที่ความเร็ว/ความเสถียร/ขนาดข้อมูล):
  • แก้ระบบตรง ๆ: กำจัดตัวแปร (ที่โชว์) · เมทริกซ์อินเวอร์ส β=(XᵀX)⁻¹Xᵀy · Cramer's rule · LU / Cholesky (เร็ว เพราะ XᵀX สมมาตรบวกแน่นอน)
  • ไม่ต้องสร้าง XᵀX (เสถียรกว่า — ที่ numpy ใช้จริง): QR decomposition · SVD
  • วิธีวนซ้ำ สำหรับข้อมูลมหาศาล: Gradient Descent — เดินลงเนินให้ SSE เล็กลงทีละก้าว · นี่คือวิธีที่ neural network / deep learning ใช้เทรนโมเดล!

ได้ a=1, b=0, c=1 เท่ากับวิธีกำจัดตัวแปรเป๊ะ · แต่การหา (XᵀX)⁻¹ 3×3 ด้วยมือ (det + adjugate) ยุ่งกว่า จึงนิยมกำจัดตัวแปร/ให้คอมพิวเตอร์ทำ
y = 1 + x²

แทนกลับกับข้อมูล: x = 3 → 1 + 3² = 10 ✓ · x = 5 → 1 + 5² = 26 ✓ — ข้อมูลชุดนี้วางบนพาราโบลาพอดี (error = 0) · ถ้ามี noise ดูข้อ 9

10. ทำไมเรียก Polynomial Regression ทั้งที่ยังใช้ Linear Regression?

จุดที่หลายคนสับสน: สมการมี x², x³ ดูเหมือนไม่เป็นเส้นตรง — แต่จริง ๆ ยังเป็น Linear Regression ในแง่ของพารามิเตอร์ (linear in parameters)

y = β₀ + β₁x + β₂x² + β₃x³

ถึงตัวแปร x จะถูกยกกำลัง แต่ พารามิเตอร์ β₀, β₁, β₂, β₃ ไม่ได้คูณกันเองหรือยกกำลัง — มันยังปรากฏแบบเชิงเส้น จึงใช้วิธี Least Squares เดียวกับ Linear Regression เพื่อหาค่าสัมประสิทธิ์ได้เลย

สรุป — Polynomial Regression มีแนวคิดดังนี้

  1. เลือกดีกรีของพหุนาม (เช่น 2, 3 หรือ 4)
  2. สร้างตัวแปรใหม่ (polynomial features): x², x³, …
  3. สร้าง Design Matrix X จากข้อมูล
  4. ใช้ Least Squares (β = (XᵀX)⁻¹Xᵀy) หาค่าสัมประสิทธิ์
  5. นำสัมประสิทธิ์ที่ได้มาสร้างสมการพหุนามไว้พยากรณ์ค่าใหม่

กล่าวอีกนัยหนึ่ง: Polynomial Regression = แปลงข้อมูลให้เป็น polynomial features แล้วใช้ Linear Regression เพื่อหาสัมประสิทธิ์ที่ทำให้ความคลาดเคลื่อนรวมมีค่าน้อยที่สุด
