Attention & Transformer ⭐

สถาปัตยกรรมที่อยู่ข้างใน GPT, Claude, Gemini — หัวใจของ AI ยุคปัจจุบัน

🎯 เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบาย attention ได้ใน 1 ประโยค: "ทุกคำหันไปมองทุกคำ แล้วถ่วงน้ำหนักว่าจะฟังใครมากแค่ไหน"
  2. คำนวณ attention weight อย่างง่าย (dot product + softmax) บนเวกเตอร์จิ๋วได้
  3. วาดโครง Transformer block ได้: attention → MLP ซ้อนกันหลายชั้น
  4. เช็กได้ว่า Transformer ตอบ "ใบสเปก" จากสัปดาห์ที่แล้วครบทุกข้อ

ส่วนที่ 1 — แนวคิด แนวคิด

คำถามเปิดคาบ: "ตอนอ่านคำว่า 'มัน' ในประโยค 'แมวที่เลี้ยงไว้หน้าบ้านหลังเก่า...มันหิว' สมองคุณกระโดดกลับไปหาคำว่าอะไร — และรู้ได้ไงว่าต้องกระโดดไปหาคำนั้น?"

Attention: ให้ทุกคำ "หันไปมอง" ทุกคำ

เลิกบีบความจำเป็นทอดๆ แบบ RNN — attention ให้คำทุกคำมองเห็นคำอื่นทั้งประโยคพร้อมกัน แล้วคำนวณว่าควร "ฟัง" แต่ละคำหนักแค่ไหน ผ่าน 3 บทบาท (ทุกคำเล่นครบทั้งสามบทบาท):

  • Query (Q) — "ฉันกำลังตามหาอะไร" (คำว่า "มัน" ตามหา: สิ่งมีชีวิตที่ถูกพูดถึงก่อนหน้า)
  • Key (K) — "ป้ายชื่อของฉันคืออะไร" (คำว่า "แมว" มีป้าย: สัตว์/ประธานของเรื่อง)
  • Value (V) — "ถ้าเลือกฟังฉัน ฉันจะส่งข้อมูลอะไรให้"

ขั้นตอน: เอา Q ของคำเราไป dot product กับ K ของทุกคำ (วัดความเข้ากัน) → softmax (ได้น้ำหนักรวมเป็น 1 — เครื่องมือจากเรื่อง Neural Network!) → เฉลี่ย V ของทุกคำตามน้ำหนัก = ความหมายใหม่ของคำเราที่ "ดูดบริบท" มาแล้ว

Transformer = (Attention + MLP) ซ้อนกันหลายสิบชั้น

หนึ่ง Transformer block: attention (ให้คำคุยกัน) ตามด้วย MLP (ให้แต่ละคำครุ่นคิดกับตัวเอง — เรื่อง Neural Network!) ซ้อนหลายสิบชั้น ชั้นล่างจับไวยากรณ์ใกล้ตัว ชั้นสูงจับความหมายเชิงเรื่องราว + ทำ attention หลายชุดพร้อมกัน (multi-head — แต่ละ head สนใจคนละแง่) + ฝังลำดับตำแหน่งคำ (positional encoding) เพราะลำพัง attention ไม่รู้ว่าใครมาก่อนหลัง

เช็กใบสเปกจากสัปดาห์ที่แล้ว

สเปกที่ต้องการTransformer ตอบโจทย์ด้วย
จำระยะไกลได้ ไม่มีคอขวดทุกคำมองทุกคำโดยตรง — ระยะทาง 1 ก้าวเสมอ ไม่ว่าห่างกี่พันคำ
เทรนขนานบน GPU ได้ทุกคำคำนวณพร้อมกัน ไม่ต้องรอทีละคำแบบ RNN
generalize ได้ ไม่ใช่ตารางนับทุกอย่างคือ weight ที่เทรนด้วย backprop — เครื่องมือเดียวกับเรื่อง Backpropagation

ราคาที่จ่าย: ทุกคำมองทุกคำ = งานโตแบบ n² ตามความยาวบริบท — นี่คือเหตุผลที่ "context window" ของ LLM มีขีดจำกัด และทำไมบริบทยาวถึงแพง

👣 เข้าใจทีละขั้น — คำนวณ attention ด้วยมือใน 6 ขั้น

ใช้เครื่องมือที่มีครบแล้ว: dot product (คูณ+บวก) กับ softmax (เรื่อง Neural Network) — จบแล้วคุณจะคำนวณหัวใจของ GPT ได้บนกระดาษ

  1. ตั้งโจทย์จิ๋ว ประโยค 3 คำ: "แมว ไล่ มัน" — เราจะคำนวณว่าคำว่า "มัน" ควรหันไปฟังใคร (คำตอบที่ควรได้: ฟัง "แมว" หนักสุด เพราะ "มัน" อ้างถึงแมว)
  2. แจกบทบาทเป็นเวกเตอร์จิ๋ว 2 มิติ Q ของ "มัน" = (2, 0) — อ่านว่า "กำลังตามหา: สิ่งมีชีวิตที่ถูกพูดถึง"
    K (ป้ายชื่อ) ของแต่ละคำ: แมว = (1, 0) "ฉันคือสิ่งมีชีวิต/ประธาน", ไล่ = (0, 1) "ฉันคือการกระทำ", มัน = (0.5, 0)
    (ของจริงเวกเตอร์ยาว 128+ มิติ และตัวเลขมาจากการเทรน — เราย่อเหลือ 2 มิติให้คิดมือได้)
  3. วัดความเข้ากันด้วย dot product คูณตำแหน่งต่อตำแหน่งแล้วบวก: Q·K(แมว) = 2×1 + 0×0 = 2 | Q·K(ไล่) = 0 + 0 = 0 | Q·K(มัน) = 1
    ทิศตรงกัน = คะแนนสูง — "สิ่งที่ตามหา" ของมัน ตรงกับ "ป้ายชื่อ" ของแมวพอดี
  4. แปลงคะแนนเป็นน้ำหนักด้วย softmax (เพื่อนเก่าเรื่อง Neural Network!) e² = 7.39, e⁰ = 1, e¹ = 2.72 → รวม = 11.11
    น้ำหนัก: แมว 67%, ไล่ 9%, มัน 24% — "มัน" ตัดสินใจฟังแมวสองในสาม
    ตัวเลขชุดนี้คุ้นไหม? คือโจทย์ softmax เดียวกับเรื่อง Neural Network เป๊ะ — ลองเปลี่ยน K ของ "ไล่" เป็น (1,1) แล้วคำนวณใหม่
  5. ผสมความหมายตามน้ำหนัก แต่ละคำมี V (ข้อมูลที่จะส่งให้) — ความหมายใหม่ของ "มัน" = 0.67×V(แมว) + 0.09×V(ไล่) + 0.24×V(มัน)
    ผลคือเวกเตอร์ของ "มัน" ตอนนี้ชุ่มไปด้วยความหมายของแมว — มันรู้แล้วว่าตัวเองหมายถึงอะไร นี่คือสิ่งที่ RNN ทำไม่ได้: ดูดความหมายจากคำที่ห่างแค่ไหนก็ได้ ใน 1 ก้าว
  6. ขยายเป็น Transformer เต็มตัว ที่ทำมา = attention ของคำเดียว 1 หัว 1 ชั้น — ของจริงทำทุกคำพร้อมกัน (ขนานบน GPU ได้!) × 12 หัว (แต่ละหัวสนใจคนละแง่) × หลายสิบชั้น สลับกับ MLP (เรื่อง Neural Network) — ทั้งหมดนี้เทรนด้วย backprop (เรื่อง Backpropagation) ไม่มีเครื่องมือใหม่เลยสักชิ้น
    ไปที่ demo "คำนวณ attention เองด้วยเวกเตอร์จิ๋ว" ข้างล่าง ใส่ตัวเลขจากขั้น 2 แล้วเช็คกับมือ

🧩 ตัวอย่างหลายแบบ — attention ทำงานหลายหน้าที่

ตัวอย่าง 1 — หากรรมของกริยา: "ฉัน กิน ข้าว"

คำว่า "กิน" ตามหากรรม: Q(กิน) = (0, 2) | ป้ายชื่อ K: ฉัน = (1, 0), กิน = (0.5, 0.5), ข้าว = (0, 1) — คำนวณ dot product, softmax แล้วสรุปว่า "กิน" ฟังใคร

ขั้น 1 — dot product ทีละคู่: Q·K(ฉัน) = (0)(1)+(2)(0) = 0 | Q·K(กิน) = (0)(0.5)+(2)(0.5) = 1 | Q·K(ข้าว) = (0)(0)+(2)(1) = 2
ขั้น 2 — ยกกำลัง e: e⁰ = 1, e¹ = 2.72, e² = 7.39 → รวม 11.11
ขั้น 3 — หารเป็นน้ำหนัก: ฉัน 9% | กิน 24% | ข้าว 67%
✅ คำตอบ: "กิน" ดูดความหมายจาก "ข้าว" หนักสุด — ตัวเลขเดียวกับบทเรียนเป๊ะแต่คนละความสัมพันธ์ (กริยา→กรรม แทน สรรพนาม→เจ้าของ) — ของจริง attention หลาย head แบ่งงานจับคนละความสัมพันธ์แบบนี้
ตัวอย่าง 2 — แก้คำกำกวม: "ตา" สองความหมาย

ประโยค ก: "ตาของฉันมองไม่ค่อยชัด" | ประโยค ข: "ตาของฉันอยู่ต่างจังหวัด" — ไม่ต้องคำนวณ ตอบเชิงกลไก: attention ของคำว่า "ตา" ในสองประโยคต่างกันยังไง แล้วเกิดอะไรกับความหมายมัน

ขั้น 1 — จุดตั้งต้นเหมือนกัน: คำว่า "ตา" ทั้งสองประโยคเริ่มจาก embedding ตัวเดียวกันเป๊ะ (ผสมทั้งความหมายอวัยวะและญาติ)
ขั้น 2 — ประโยค ก: Q ของ "ตา" จับคู่แรงกับ K ของ "มอง/ชัด" → ดูด V ฝั่งการมองเห็น → เวกเตอร์เอียงไปทาง "อวัยวะ"
ขั้น 3 — ประโยค ข: จับคู่แรงกับ K ของ "อยู่/ต่างจังหวัด" → ดูด V ฝั่งบุคคล/สถานที่ → เอียงไปทาง "คุณตา (ญาติ)"
✅ คำตอบ: คำเดียวกัน จุดเริ่มเดียวกัน แต่หลัง attention ความหมายแยกทางตามบริบท — สิ่งที่พจนานุกรม (และ embedding เดี่ยวๆ) ทำไม่ได้ นี่คือเหตุผลการมีอยู่ของ attention
ตัวอย่าง 3 — causal mask: ใครแอบดูเฉลยได้บ้าง

ตอนเทรน GPT กับประโยค 4 คำ "แมว ชอบ กิน ปลา" — เขียนตารางว่าแต่ละคำ "มองเห็น" คำไหนได้บ้าง แล้วตอบว่าทำไมต้องห้าม

ขั้น 1 — ทวนหน้าที่ตอนเทรน: ทุกตำแหน่งทำนาย "คำถัดไป" พร้อมกัน: "แมว" ทาย "ชอบ", "ชอบ" ทาย "กิน", "กิน" ทาย "ปลา"
ขั้น 2 — วางกฎการมอง: แต่ละคำมองได้เฉพาะตัวเองกับคำก่อนหน้า: แมว→{แมว} | ชอบ→{แมว, ชอบ} | กิน→{แมว, ชอบ, กิน} | ปลา→เห็นครบ
ขั้น 3 — ทดลองยกเลิกกฎ: ถ้า "กิน" มองเห็น "ปลา" ได้ มันแค่ลอกคำตอบมาตอบ — ได้คะแนนเต็มโดยไม่ได้เรียนรู้อะไร
✅ คำตอบ: ตารางสามเหลี่ยม (มองได้เฉพาะอดีต) — เหตุผลคือกันแอบดูเฉลย: data leakage เวอร์ชัน attention เรื่อง Overfitting หลอนกลับมาอีกครั้ง
ตัวอย่าง 4 — Query กำกวม: ฟังครึ่งๆ

Q = (1, 1) ป้ายชื่อสองคำ: K(a) = (1, 0), K(b) = (0, 1) — คำนวณน้ำหนัก attention

ขั้น 1 — dot product: Q·K(a) = (1)(1)+(1)(0) = 1 | Q·K(b) = (1)(0)+(1)(1) = 1 — เสมอกัน
ขั้น 2 — softmax: e¹ ต่อ e¹ → 50% / 50%
ขั้น 3 — ตีความ: Q ชี้ทแยง = "ตามหาสองอย่างพอๆ กัน" → ความหมายที่ได้คือส่วนผสมครึ่งต่อครึ่งของ V ทั้งสอง
✅ คำตอบ: 50/50 — attention ไม่ถูกบังคับให้เลือกข้าง มันผสมได้ตามสัดส่วน ความนุ่มนวลนี้ทำให้ความหมายเฉดกลางๆ ("ค่อนข้างเห็นด้วย", ประชดนิดๆ) เป็นไปได้
ตัวอย่าง 5 — ทำไมต้องหาร √d: ดูตัวเลขเอง

เวกเตอร์ยาวขึ้น dot product ก็โตขึ้น สมมุติได้คะแนน (10, 0) — softmax แล้วเป็นยังไง? เทียบกับหลังหาร √100 = 10 เหลือ (1, 0)

ขั้น 1 — softmax ของ (10, 0): e¹⁰ ≈ 22,026 ต่อ e⁰ = 1 → 99.995% / 0.005% — "อิ่มตัว" ฟังคำเดียวสนิท
ขั้น 2 — ดูผลต่อการเทรน: น้ำหนักที่เกือบ 0/1 สนิท ทำให้ gradient ผ่าน softmax แทบเป็นศูนย์ → โมเดลปรับตัวไม่ได้ "หูดับ" ตั้งแต่เกิด
ขั้น 3 — หารด้วย √d = √100 = 10: คะแนนเหลือ (1, 0) → softmax = 2.72 ต่อ 1 → 73% / 27% — เอียงแต่ยังเปิดหู ✓
✅ คำตอบ: ก่อนหาร 99.995/0.005 หลังหาร 73/27 — ÷√d ไม่ใช่เครื่องประดับในสูตร มันคือตัวกันโมเดลหูดับตั้งแต่เริ่มเทรน
ตัวอย่าง 6 — multi-head: หลายหัว หลายหน้าที่

ประโยค "แมวดำที่บ้านยายเลี้ยงมันหิว" — head 1 จับไวยากรณ์ (คำขยาย→คำหลัก), head 2 จับการอ้างถึง — แต่ละ head ของคำว่า "ดำ" และ "มัน" มองไปที่ไหน?

ขั้น 1 — head 1 (ไวยากรณ์) มองคำว่า "ดำ": Q ของมันถามว่า "ฉันขยายใคร" → จับคู่กับ K ของคำนามใกล้ๆ → "แมว"
ขั้น 2 — head 2 (การอ้างถึง) มองคำว่า "มัน": Q ถามว่า "ฉันแทนใคร" → จับคู่กับ K ของประธานเรื่อง → "แมว" เช่นกัน
ขั้น 3 — สังเกตความต่าง: เป้าเดียวกัน แต่คนละเหตุผล — สองความสัมพันธ์นี้ต้องการ Q/K คนละแบบ จึงต้องคนละ head
✅ คำตอบ: ทั้งคู่ชี้ไป "แมว" ด้วยเหตุผลต่างกัน — และไม่มีใครสั่งให้ head ไหนทำหน้าที่อะไร การแบ่งงานเกิดเองจากการเทรน (นักวิจัยค้นพบทีหลังด้วยการ "ผ่า" โมเดล — สาขา interpretability)
ตัวอย่าง 7 — ทำไมต้องบอกตำแหน่งคำ (positional encoding)

"หมากัดคน" กับ "คนกัดหมา" — ถ้าไม่มี positional encoding, attention เห็นสองประโยคนี้ต่างกันไหม?

ขั้น 1 — ดูว่า attention รับอะไรเข้าไป: เซตของเวกเตอร์คำ {หมา, กัด, คน} — การคำนวณ Q·K ทุกคู่ไม่มีตรงไหนใช้ "ลำดับ" เลย
ขั้น 2 — เทียบสองประโยค: "หมากัดคน" กับ "คนกัดหมา" ให้เซตเดียวกันเป๊ะ → attention เพียวๆ เห็นไม่ต่างเลย (เหมือนถุงคำ)
ขั้น 3 — วิธีแก้: ฝัง "เลขที่นั่ง" (ตำแหน่ง 1, 2, 3...) เข้าไปในเวกเตอร์แต่ละคำก่อนเข้า attention → หมา@1 กับ หมา@3 กลายเป็นเวกเตอร์คนละตัว
✅ คำตอบ: ไม่ต่าง จึงต้องมี positional encoding — กฎจำ: ภาษาไหนสลับคำแล้วความหมายเปลี่ยน ภาษานั้นต้องการมัน (ภาษาไทยคือตัวอย่างชั้นดี)
ตัวอย่าง 8 — ราคาของ n²: ทำไมบริบทยาวถึงแพง

attention ให้ทุกคำมองทุกคำ — นับจำนวนคู่: บริบท 1,000 token เทียบกับ 100,000 token งานต่างกันกี่เท่า?

ขั้น 1 — นับคู่กรณีสั้น: ทุกคำมองทุกคำ: 1,000 × 1,000 = 1 ล้านคู่
ขั้น 2 — นับคู่กรณียาว: 100,000 × 100,000 = 10,000 ล้านคู่
ขั้น 3 — เทียบอัตราโต: ความยาว ×100 แต่งาน ×10,000 — โตเป็นกำลังสอง (n²) ไม่ใช่เชิงเส้น
✅ คำตอบ: 10,000 เท่า — นี่คือเหตุผลที่ context window มีขีดจำกัดและคิดเงินตามความยาว งานวิจัยลด n² (sparse/linear attention) คือสมรภูมิใหญ่ของวงการตอนนี้
ตัวอย่าง 9 — attention มองรูปภาพ: Vision Transformer

ถ้า "คำ" ของภาพคือชิ้นภาพ 16×16 pixel (patch) — attention ของ patch "หางหมา" จะหันไปมองอะไร แล้วมันต่างจาก CNN ยังไง?

ขั้น 1 — แปลงภาพเป็น "ประโยค": หั่นภาพเป็น patch 16×16 → แต่ละ patch = หนึ่ง token เรียงเป็นลำดับเหมือนคำ
ขั้น 2 — รัน attention ตามปกติ: patch "หางหมา" สร้าง Q ตามหา "ส่วนอื่นของสิ่งเดียวกัน" → จับคู่ K ของ patch "หัวหมา/ตัวหมา" ได้ทันทีใน 1 ชั้น แม้อยู่คนละมุมภาพ
ขั้น 3 — เทียบ CNN: kernel เห็นแค่ 3×3 — กว่า receptive field จะกว้างถึงสองมุมภาพ ต้องซ้อนหลายสิบชั้น
✅ คำตอบ: มอง "ชิ้นส่วนเดียวกันที่อยู่ไกล" ได้ใน 1 ก้าว — ViT ใช้สมการ attention เดียวกับภาษาทุกบรรทัด แค่เปลี่ยน token เป็น patch สถาปัตยกรรมเดียวครองสองโลก = เหตุที่โมเดลยุคนี้ multimodal ได้ง่าย (เรื่อง Diffusion)
ตัวอย่าง 10 — attention map ใช้ "อ่านใจ" โมเดลได้ แต่ระวัง

เครื่องมือ debug ของวิศวกร: เปิดดูว่าตอนแปลคำนี้ผิด โมเดล attend ไปที่คำไหน — เชื่อถือได้ 100% ไหม?

ขั้น 1 — วิธีใช้เชิงบวก: โมเดลแปล "ตา" ผิด → เปิด attention map → เห็นว่ามัน attend ไปคำหลอก → รู้จุดแก้ (ข้อมูล/prompt) ตรงเป้า — มีประโยชน์จริง
ขั้น 2 — เหตุผลที่เชื่อ 100% ไม่ได้: น้ำหนัก attention ชั้นนี้ อาจถูกกลบ/ชดเชยโดยชั้นถัดไป — attention สูงไม่ได้แปลว่าข้อมูลนั้น "ถูกใช้" ตัดสินคำตอบจริง
ขั้น 3 — หลักฐานจากวงการ: งานวิจัยชื่อดัง "Attention is not Explanation" (2019) แสดงเคสที่เปลี่ยนน้ำหนัก attention แล้วคำตอบไม่เปลี่ยน
✅ คำตอบ: เชื่อถือได้ระดับ "เบาะแส" ไม่ใช่ "คำพิพากษา" — ใช้ชี้จุดสืบสวนต่อ อย่าใช้ปิดคดี

ส่วนที่ 2 — ทดลองเล่น ทดลองเล่น

🎬 แอนิเมชัน: "ทุกคำหันไปมองทุกคำ" พร้อมกันทั้งประโยค

นี่คือสิ่งที่เกิดใน attention หนึ่งชั้น — เส้นโค้งคือ "สายตา" ของแต่ละคำ ความหนาของเส้น = น้ำหนักที่ฟังคำนั้น สังเกตตอนถึงคำว่า "มัน": สายตาพุ่งกลับไปหา "แมว" ข้ามคำอื่นทั้งหมด

🎬 Attention ทีละจังหวะ — คำนวณจริงทุกตัวเลข (ชิ้นเอกของทั้งวิชา)

ประโยคจิ๋ว "แมว ไล่ หนู" — แต่ละคำมี embedding 2 มิติ แล้วใช้เมทริกซ์ weight จริง 3 ชุดข้างล่างสร้าง Q, K, V จากนั้นเดินดู 4 จังหวะของ attention แบบเห็นเลขทุกตัวถูกคูณจริง: ① ทุกคำสร้าง Q K V (embedding × เมทริกซ์) → ② เอา Q ของคำที่เลือกไปทาบกับ K ทุกคำ (dot product) → ③ หาร √d แล้ว softmax เป็นน้ำหนัก → ④ ถัวเฉลี่ย V ตามน้ำหนัก = ความหมายใหม่ · ตัวเลขชุดเดียวกันนี้จะไปโผล่ในโค้ด numpy ส่วนที่ 3 ให้รันเช็กได้เอง

ส่วนที่ 3 — โค้ดจริง: attention 25 บรรทัด รันใน Colab ได้เลย ลงมือ

นี่คือ attention หนึ่งหัวแบบครบสมการ ใช้ตัวเลขชุดเดียวกับแอนิเมชัน 🎬 ข้างบนเป๊ะทุกตัว — เปิด Google Colab → New Notebook → วาง → Run แล้วเทียบ print กับตัวเลขบนจอ: Q, K, V, คะแนน, น้ำหนัก softmax, เวกเตอร์ผลลัพธ์ ต้องตรงกันหมด

import numpy as np

# ประโยค "แมว ไล่ หนู" — embedding 2 มิติ (เลขเดียวกับแอนิเมชันข้างบน)
X = np.array([[1.0, 0.0],    # แมว
              [0.0, 1.0],    # ไล่
              [1.0, 0.5]])   # หนู

W_Q = np.array([[1.0, 2.0], [2.0, 0.0]])   # เมทริกซ์สร้าง Query "คำถาม"
W_K = np.array([[1.0, 0.0], [0.5, 1.0]])   # เมทริกซ์สร้าง Key "ป้ายชื่อ"
W_V = np.array([[1.0, 0.5], [0.0, 1.0]])   # เมทริกซ์สร้าง Value "ของที่ส่งให้"

Q, K, V = X @ W_Q, X @ W_K, X @ W_V        # ทุกคำสร้าง 3 บทบาทพร้อมกัน (จังหวะ 1)
print("Q =\n", Q, "\nK =\n", K, "\nV =\n", V)

scores = Q @ K.T / np.sqrt(K.shape[1])     # dot product ทุกคู่ ÷ √d (จังหวะ 2-3)
print("คะแนนหลังหาร √2 =\n", scores.round(2))

def softmax(z):                            # เพื่อนเก่าจากเรื่อง Neural Network
    e = np.exp(z - z.max(axis=-1, keepdims=True))
    return e / e.sum(axis=-1, keepdims=True)

A = softmax(scores)                        # น้ำหนัก attention — แต่ละแถวรวม = 1
print("attention weights =\n", A.round(3)) # แถว "ไล่" ต้องได้ [0.343, 0.169, 0.488]

out = A @ V                                # ถัวเฉลี่ย V ตามน้ำหนัก (จังหวะ 4)
print("ความหมายใหม่ =\n", out.round(3))     # แถว "ไล่" ต้องได้ [0.831, 0.829]

แล้วดูว่าโค้ดเดียวกันเป๊ะทำงานที่สเกลจริงได้เลย — แค่เปลี่ยนตัวเลขให้ใหญ่ขึ้น:

# สเกลจริง: ประโยค 100 token, เวกเตอร์ 512 มิติ — โค้ดบรรทัดเดิมทุกบรรทัด
d = 512
X = np.random.randn(100, d)
W_Q, W_K, W_V = [np.random.randn(d, d) * 0.05 for _ in range(3)]
Q, K, V = X @ W_Q, X @ W_K, X @ W_V
A = softmax(Q @ K.T / np.sqrt(d))          # ตาราง attention 100×100 (นี่ไง n²!)
out = A @ V
print(A.shape, out.shape)                  # (100, 100) (100, 512)

สิ่งที่ต้องสังเกต:

  • แถว "ไล่" ของ A คือแท่งน้ำหนักในจังหวะ 3 ของแอนิเมชันเป๊ะ (0.343 / 0.169 / 0.488) — แอนิเมชันไม่ได้โกหก และหัวใจของ GPT เขียนได้ใน ~10 บรรทัดจริงๆ
  • แต่ละแถวของ A รวมได้ 1.0 เสมอ (ลอง A.sum(axis=1)) — ทุกคำแบ่ง "หูฟัง" ของตัวเอง 100% ให้ทั้งประโยค
  • บล็อกที่สอง: ตาราง A ขนาด 100×100 โตเป็น n² ตามความยาวประโยค — ลองเปลี่ยน 100 เป็น 2000 แล้วจับเวลา จะเข้าใจราคาของ context ยาวทันที

ส่วนที่ 4 — แบบฝึกหัด แบบฝึกหัด

📝 ข้อ 1 (ทำในคาบ) — attention ด้วยมือ

Q = [1, 0] และ K ของ 3 คำ: K(แมว) = [2, 0], K(เสื่อ) = [0, 1], K(กิน) = [−1, 0] จงคำนวณ (ก) dot product ทั้งสาม (ข) softmax เป็น attention weight (ใช้ e² ≈ 7.39, e⁰ = 1, e⁻¹ ≈ 0.37) (ค) สรุป: "มัน" ฟังใครหนักสุด

ดูเฉลย (ก) แมว: 1×2+0×0 = 2 | เสื่อ: 0 | กิน: −1
(ข) ผลรวม e = 7.39 + 1 + 0.37 = 8.76 → แมว 84%, เสื่อ 11%, กิน 4%
(ค) "มัน" ดูดความหมายจาก "แมว" มาเกือบทั้งหมด — โมเดลแก้ปัญหาสรรพนามด้วยคณิตศาสตร์ล้วนๆ
📝 ข้อ 2 (ทำในคาบ) — งานโต n²

ถ้า attention ต้องให้ทุก token มองทุก token จงหาจำนวน "คู่ที่ต้องคำนวณ" เมื่อบริบทยาว 100, 1,000, 100,000 token แล้วตอบ: ทำไมราคาค่าใช้ LLM ถึงคิดตามจำนวน token และทำไมแชทยาวๆ ถึงช้าลง

ดูเฉลย 100² = 10⁴ | 1,000² = 10⁶ | 100,000² = 10¹⁰ — ยาวขึ้น 1,000 เท่า งานโต 1,000,000 เท่า ไม่ใช่เชิงเส้น จึงต้องจำกัด context window และคิดเงินตาม token ที่ประมวลผล
🏠 ข้อ 3 (การบ้าน) — รายงาน Transformer Explainer

จาก Transformer Explainer: ป้อนประโยคภาษาอังกฤษที่มีสรรพนามกำกวม เช่น "The cat chased the mouse because it was..." แคปภาพ attention map แล้วรายงานว่า head ไหน/ชั้นไหนที่ "it" มองกลับไปหาคำนาม พร้อมความเห็น 3–4 ประโยค

🏠 ข้อ 4 (การบ้าน) — ตรวจใบสเปกของตัวเอง

เอา "ใบสเปกสถาปัตยกรรมในฝัน" ที่เขียนไว้สัปดาห์ที่แล้ว มาเทียบกับ Transformer ทีละข้อ: ข้อไหนตอบครบ ข้อไหนยังไม่ตอบ (เช่น ราคา n²?) — สรุปครึ่งหน้า

ส่วนที่ 5 — ขั้นบันไดของสัปดาห์นี้ ขั้นบันได

🪜 สัปดาห์นี้พาเราเข้าใกล้ LLM อย่างไร:
เราเพิ่งเรียน "ชิ้นส่วนสุดท้าย" — attention ที่ให้ทุกคำมองทุกคำ + MLP เดิม ซ้อนเป็น Transformer ตอนนี้เรามีครบ: โมเดล (Transformer) + งาน (ทำนาย token ถัดไป) + วิธีเทรน (backprop) + อาหาร (ข้อความมหาศาล)
สัปดาห์หน้า: 🏁 ประกอบทุกชิ้นเป็น LLM ตัวจริง — แล้วตอบคำถามแรกของเทอมให้จบ: "ChatGPT ฉลาดเพราะอะไร และทำไมบางทีมันโกหก"
