SVM: หา "เส้นแบ่งที่กว้างที่สุด" (และเวทมนตร์ Kernel)

สัปดาห์ก่อนต้นไม้แบ่งข้อมูลด้วยการถามทีละเงื่อนไข — สัปดาห์นี้เราถามคำถามใหม่ที่คมกว่า: ในบรรดา "เส้นแบ่ง" นับไม่ถ้วนที่แยกสองกลุ่มได้ เส้นไหนดีที่สุด? คำตอบของ SVM คือ "เส้นที่ทิ้งช่องว่างกว้างที่สุด" — แนวคิดที่ครองวงการก่อนยุค deep learning

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายได้ว่า SVM เลือกเส้นแบ่งด้วยหลัก margin สูงสุด — เส้นที่อยู่ "กลางถนน" ที่กว้างที่สุดระหว่างสองกลุ่ม เพราะมันเผื่อความคลาดเคลื่อนได้มากที่สุด
  2. รู้จัก support vectors — จุดไม่กี่จุดที่แตะขอบถนน ซึ่ง เพียงลำพัง ก็กำหนดเส้นได้ทั้งเส้น (จุดอื่นถอดออกก็ไม่เปลี่ยน)
  3. เข้าใจ soft margin (ปุ่ม C): ยอมให้ผิดบ้างเพื่อถนนกว้าง (C น้อย) หรือดันให้ถูกทุกจุดแต่ถนนแคบ (C มาก)
  4. เข้าใจ Kernel trick: ข้อมูลที่ 2 มิติแยกด้วยเส้นตรงไม่ได้ พอ "ยกขึ้นมิติที่สูงกว่า" กลับใช้ระนาบแบนตัดได้ทันที
  5. วาง SVM ลงในไทม์ไลน์ AI ได้ — แชมป์ยุค ~1995–2010 ก่อน deep learning จะมาแทน แล้วเชื่อมสู่ Neural Network สัปดาห์หน้า

ส่วนที่ 1 — เส้นไหน "ดีที่สุด"? แนวคิด

สองกลุ่มนี้แยกกันด้วยเส้นตรงได้สบาย — แต่ปัญหาคือ เส้นที่แยกได้มีนับไม่ถ้วน! เอียงนิด เลื่อนหน่อยก็ยังแยกถูก แล้วจะเลือกเส้นไหนดี? สัญชาตญาณของ SVM: เส้นที่ดีที่สุดคือเส้นที่วาง "ถนน" (margin) กว้างที่สุด คั่นกลาง — ยิ่งถนนกว้าง เจอจุดใหม่ที่เยื้องนิดหน่อยก็ยังทายถูก = ทนทานที่สุด

ส่วนที่ 3 — โลกจริงข้อมูลปนกัน: ปุ่ม C (soft margin) ทดลองเล่น

ข้อมูลจริงมักไม่ได้แยกกันสวย ๆ — มีจุดหลงข้ามฝั่งปนอยู่ ถ้ายืนกรานหาเส้นที่แยกถูก ทุกจุด จะได้ถนนแคบเป็นเส้นด้ายและ overfit · SVM แก้ด้วย soft margin: ยอม "ปรับ" ให้บางจุดล้ำถนนหรือข้ามฝั่งได้ แลกกับถนนที่กว้างและมั่นคงกว่า — ปุ่มที่คุมการแลกเปลี่ยนนี้คือ C

ส่วนที่ 4 — เวทมนตร์ Kernel: 2 มิติแยกไม่ได้ → ยกขึ้น 3 มิติ ทีละขั้น

บางข้อมูล ไม่มีทางใช้เส้นตรงแยกได้ในระนาบ 2 มิติ เช่นกลุ่มในสุดถูกล้อมด้วยกลุ่มวงแหวนรอบนอก — ลากเส้นตรงยังไงก็ผ่ากลางกลุ่มใดกลุ่มหนึ่งเสมอ · แต่ SVM มีไพ่ตาย: "ยก" ข้อมูลขึ้นมิติที่สูงกว่า แล้วในมิติใหม่นั้น ระนาบแบน ๆ กลับตัดแยกได้พอดี นี่คือ Kernel Trick — กดไล่ 3 ฉากแล้วลากหมุนดูเองได้

โอ้โหโมเมนต์: ยกวงแหวนขึ้น แล้วเฉือนด้วยระนาบเดียว

ฉาก 1: มองจากบน — แกนกลาง ถูก วงแหวน ล้อม เส้นตรงแยกไม่ได้ · ฉาก 2: ยกความสูง z = x²+y² (จุดยิ่งไกลศูนย์ยิ่งลอยสูง) · ฉาก 3: ตอนนี้ ระนาบแนวนอนแผ่นเดียว เฉือนแยกได้สบาย!

ลากเพื่อหมุน
ฉาก 1 มองจากบน (แยกไม่ได้) · ฉาก 2 ยกมิติ z=x²+y² · ฉาก 3 ระนาบเฉือนแยก

สรุปกลไก: "ยกขึ้นมิติสูง → เฉือนด้วยระนาบแบน" คือหัวใจของ Kernel Trick · ความฉลาดจริงคือ SVM ไม่ต้องคำนวณพิกัด 3 มิติจริง ๆ เลย — มันใช้ kernel function (เช่น RBF) คำนวณ "ความคล้าย" ของคู่จุดในมิติสูงโดยตรง เหมือนได้ผลของการยกมิติมาฟรี ๆ · เส้นแบ่งโค้ง ๆ ที่เห็นตอนมองกลับ 2 มิติ = ร่องรอยของระนาบแบนในมิติที่สูงกว่า

โค้ดจริง — เล่นบน Google Colab ได้เลย (scikit-learn)

เปรียบเทียบ kernel เชิงเส้น (linear) กับ RBF บนข้อมูลวงแหวน make_circles แล้ววาดเส้นแบ่งพร้อมนับ support vectors · กด คัดลอก แล้ววางใน Colab

# SVM บน scikit-learn — เทียบ kernel linear กับ rbf บนข้อมูลวงแหวน
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_circles
from sklearn.svm import SVC

# 1) สร้างข้อมูลวงแหวน: กลุ่มในสุด vs วงรอบนอก (แยกด้วยเส้นตรงไม่ได้)
X, y = make_circles(n_samples=200, factor=0.4, noise=0.10, random_state=0)

# 2) เทรนสองโมเดล: เส้นตรง (linear) กับ ยกมิติด้วย kernel RBF
lin = SVC(kernel='linear', C=1.0).fit(X, y)
rbf = SVC(kernel='rbf', C=1.0, gamma=1.0).fit(X, y)

print('linear  แม่นยำ:', lin.score(X, y), ' | support vectors:', len(lin.support_))
print('rbf     แม่นยำ:', rbf.score(X, y), ' | support vectors:', len(rbf.support_))
# linear จะได้ ~50% (แยกวงแหวนไม่ได้) ส่วน rbf จะเกือบ 100%

# 3) วาดเส้นแบ่ง (decision boundary) ของแต่ละ kernel
def plot_boundary(model, title):
    xx, yy = np.meshgrid(np.linspace(-1.5, 1.5, 300),
                         np.linspace(-1.5, 1.5, 300))
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)
    plt.contourf(xx, yy, Z, alpha=0.25, cmap='coolwarm')
    plt.scatter(X[:, 0], X[:, 1], c=y, cmap='coolwarm', edgecolors='k', s=28)
    # วงกลมรอบ support vectors
    sv = model.support_vectors_
    plt.scatter(sv[:, 0], sv[:, 1], s=140, facecolors='none', edgecolors='lime')
    plt.title(title); plt.gca().set_aspect('equal')

plt.figure(figsize=(10, 4))
plt.subplot(1, 2, 1); plot_boundary(lin, 'kernel = linear (แยกวงแหวนไม่ได้)')
plt.subplot(1, 2, 2); plot_boundary(rbf, 'kernel = rbf (ยกมิติ → เฉือนได้)')
plt.tight_layout(); plt.show()

# ลองปรับเล่น: C=100 (ถนนแคบ ดันให้ถูกทุกจุด) กับ gamma=5 (โค้งตามจุดมากขึ้น)

ส่วนที่ 6 — แบบฝึกหัด แบบฝึกหัด

ข้อ 1 — support vectors คืออะไร

อธิบายด้วยคำพูดตัวเองว่า "support vectors" คือจุดแบบไหน และทำไมถ้าลบจุดที่ ไม่ใช่ support vector ทิ้ง เส้นแบ่งของ SVM ถึงไม่เปลี่ยน

ดูเฉลย

ขั้น 1 — support vectors คือจุดที่อยู่ "แตะขอบถนน" พอดี (จุดที่ใกล้เส้นตัดสินที่สุดของแต่ละฝั่ง)

ขั้น 2 — SVM หาเส้นที่กว้างสุดโดยดูแค่จุดที่ประชิดถนนสองข้าง — จุดที่อยู่ลึกในดินแดนตัวเองไม่มีผลต่อ "ความกว้างสุด"

✅ คำตอบ: เส้นถูกกำหนดโดย support vectors เท่านั้น ลบจุดอื่นทิ้งกี่จุดก็ได้ เส้นไม่ขยับ — นี่คือเหตุผลที่ SVM ประหยัดหน่วยความจำ (จำแค่ไม่กี่จุด)
ข้อ 2 — ทำไมถนนกว้างถึง generalize ดีกว่า

มีเส้นแบ่งสองเส้นที่แยกข้อมูลฝึกถูกทั้งคู่ เส้น A ทิ้งถนนกว้าง เส้น B ทิ้งถนนแคบ — เส้นไหนน่าจะทายจุดใหม่ (ที่ยังไม่เคยเห็น) ได้แม่นกว่า เพราะอะไร

ดูเฉลย

ขั้น 1 — จุดใหม่มักเยื้องจากจุดฝึกไปนิดหน่อยเพราะ noise

ขั้น 2 — ถ้าถนนแคบ (เส้น B) จุดที่เยื้องนิดเดียวก็ตกไปอีกฝั่งของเส้น = ทายผิดง่าย

✅ คำตอบ: เส้น A (ถนนกว้าง) ดีกว่า — margin กว้างทำหน้าที่เป็น "เขตกันชน" เผื่อความคลาดเคลื่อน จุดใหม่ต้องเยื้องเยอะมากถึงจะข้ามเส้น จึงทนต่อ noise และ generalize ดีกว่า
ข้อ 3 — ปุ่ม C คุมอะไร

เพื่อนตั้ง C ไว้สูงมาก แล้วบ่นว่าโมเดลได้ accuracy บน training set 100% แต่พอใช้จริงกลับพลาดเยอะ — เกิดอะไรขึ้น และควรปรับ C ไปทางไหน

ดูเฉลย

ขั้น 1 — C สูง = ลงโทษการทายผิดหนัก → โมเดลดันหาเส้นที่แยกถูกทุกจุด ถนนแคบมาก

ขั้น 2 — ถนนแคบ = เส้นบิดตามทุก noise ใน training set = overfitting (train ดีเลิศ test พัง)

✅ คำตอบ: ควร ลด C ลง — ยอมให้ผิดบ้างในข้อมูลฝึกเพื่อได้ถนนกว้างขึ้นและ generalize ดีขึ้น (C คือปุ่มแลกเปลี่ยนระหว่าง "ถนนกว้าง" กับ "แยกถูกทุกจุด")
ข้อ 4 — เลือก linear หรือ rbf

งานสองแบบ: (ก) แยกอีเมลสแปม/ไม่สแปมด้วยฟีเจอร์นับคำหลายพันมิติ ที่มักแยกกันได้เกือบเป็นเส้นตรงอยู่แล้ว (ข) แยกข้อมูลรูปวงแหวนซ้อนกันใน 2 มิติ — แต่ละงานควรใช้ kernel อะไร เพราะอะไร

ดูเฉลย

ขั้น 1 — ข้อมูลมิติสูงมาก ๆ (เช่นนับคำ) มักแยกเชิงเส้นได้อยู่แล้ว → linear เร็วและพอเพียง ไม่ต้องยกมิติ

ขั้น 2 — วงแหวนซ้อนกันแยกด้วยเส้นตรงไม่ได้เด็ดขาด → ต้องยกมิติด้วย rbf ให้ระนาบเฉือนได้

✅ คำตอบ: (ก) ใช้ linear (มิติสูง แยกเชิงเส้นได้ เทรนเร็ว) · (ข) ใช้ rbf (เส้นแบ่งต้องโค้ง ต้องพึ่ง kernel trick) — หลักคิด: เริ่มจาก linear ก่อนเสมอ ถ้าแยกไม่ได้ค่อยขยับไป rbf

เรามาถึงไหนแล้ว:

  • Neural Network (เรื่อง Neural Network) → — แทนที่เราจะออกแบบ "การยกมิติ/ฟีเจอร์" เอง โครงข่ายประสาท เรียนรู้ feature ของมันเอง จากข้อมูล เปิดประตูสู่ deep learning
