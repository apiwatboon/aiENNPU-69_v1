Backpropagation: เครื่องจักรเรียนรู้ได้อย่างไร (คำนวณตามได้ทุกตัวเลข)

สัปดาห์ที่แล้วเราส่งตัวเลข "ไปข้างหน้า" ผ่านเครือข่ายจนได้คำตอบ — สัปดาห์นี้เรียนกลไกที่ทำให้มัน "ฉลาดขึ้น": ส่งความผิดพลาด "ย้อนกลับ" ไปบอกทุก weight ว่าต้องขยับทางไหน — และนี่คืออัลกอริทึมเดียวกันเป๊ะที่เทรน GPT

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายหลัก Gradient Descent: w ← w − η·(ความชัน) และผลของ learning rate η (เล็กไป = ช้า, ใหญ่ไป = ส่าย/ระเบิด)
  2. ใช้ chain rule คำนวณอนุพันธ์ของฟังก์ชันซ้อนด้วยตัวเลขจริงได้
  3. ไล่ Backpropagation ครบวงจร ด้วยมือ: forward → loss → backward (หา δ ทุกโหนด) → update ทุก weight
  4. อธิบายบทบาทของ σ′(z) = a(1−a) และเหตุที่เครือข่ายลึกเจอปัญหา vanishing gradient
  5. เชื่อมโยงได้ว่า GPT ก็เทรนด้วยลูปเดียวกันนี้ — ต่างแค่ขนาด

ส่วนที่ 1 — แนวคิด: คนตาบอดหาก้นหุบเขา แนวคิด

สัปดาห์ที่แล้วเราเห็นว่าเครือข่ายเป็นแค่ "เครื่องคำนวณ" ที่มีปุ่มหมุนชื่อ weight เต็มไปหมด — คำถามล้านดอลลาร์คือ ใครเป็นคนหมุนปุ่มพวกนั้นให้ถูก?

นิยามปัญหาให้ชัด: เรามี loss = ตัวเลขบอกว่า "ตอนนี้เครือข่ายตอบพลาดแค่ไหน" — งานของเราคือหาค่า weight ที่ทำให้ loss ต่ำที่สุด แต่ weight มีเป็นล้านตัว จะลองสุ่มทีละค่าก็ตายก่อน

ทางออกเปรียบเหมือน คนตาบอดยืนอยู่บนภูเขาแล้วอยากลงไปก้นหุบเขา — มองไม่เห็นทั้งแผนที่ แต่รู้สิ่งเดียว: "พื้นใต้เท้าตอนนี้เอียงไปทางไหน" (ความชัน = gradient) → ก้าวสวนทางความชันทีละก้าวเล็กๆ ซ้ำไปเรื่อยๆ เดี๋ยวก็ถึงก้นแอ่ง:

wใหม่  =  wเดิม − η × ความชันของ loss ที่จุดนั้น

ตัว η (เอตา = learning rate) คือ "ขนาดก้าว" — และ Backpropagation คือวิธีคำนวณความชันของ weight ทุกตัวพร้อมกันอย่างเร็วมหัศจรรย์ (เดี๋ยวเจอในส่วนที่ 3–4)

ส่วนที่ 2 — เวทีเล่น: ปล่อยลูกบอลกลิ้งลงหุบ loss ทดลองเล่น

⛰️ Gradient Descent Playground — เห็นทุกก้าวเป็นตัวเลข

ลากลูกบอลไปวางตรงไหนก็ได้บนโค้ง loss แล้วกด ก้าวถัดไป — ใต้กราฟจะพิมพ์เลขเต็มทุกก้าว: ความชันเท่าไหร่ คูณ η ได้เท่าไหร่ w ใหม่คือเท่าไหร่ · แล้วลอง ปุ่ม η ทั้ง 3: 0.1 เดินนิ่มถึงก้นแอ่ง · 0.9 ส่ายซ้ายขวา · 1.5 กระเด็นออกนอกหุบ!

ส่วนที่ 3 — Chain Rule: หัวใจของการส่งย้อนกลับ ทีละขั้น

เครือข่ายคือ "ฟังก์ชันซ้อนฟังก์ชัน" หลายชั้น — จะรู้ว่า weight ต้นทางกระทบ loss ปลายทางแค่ไหน ต้องใช้กฎลูกโซ่: อนุพันธ์ของฟังก์ชันซ้อน = ผลคูณของอนุพันธ์ทีละท่อน ลองกับตัวอย่างจิ๋วที่เห็นเลขครบ:

y = (2x + 1)²  ที่  x = 1   →   แตกเป็นท่อน: u = 2x+1 แล้ว y = u²

ส่วนที่ 4 — ห้องแล็บ Backprop: เครือข่าย 2-2-1 คำนวณจริงทุกตัวเลข ชิ้นเอก

เครือข่ายจริงขนาดจิ๋ว: อินพุต 2 → ซ่อน 2 (sigmoid) → เอาต์พุต 1 (sigmoid) · loss = ½(y−ŷ)² — กดไล่ดูทีละเลขในห้องแล็บด้านล่างได้เลย

🧪 กด "ขั้นถัดไป" ไล่ดูทีละขั้น — แล้วกด "เทรนรอบถัดไป" ซ้ำๆ ดู loss ไหลลง

  • ดูตารางขั้นที่ 6: weight เส้นไหนถูกขยับแรงสุด? ลองอธิบายจากสูตร grad = δ × a ว่าทำไม

ส่วนที่ 5 — ที่มาของ "สมการ weight ใหม่" ทีละขั้น: เครือข่าย 2‑2‑2 เจาะลึกทีละขั้น

ห้องแล็บด้านบนมีเอาต์พุต ตัวเดียว — แต่ของจริง (และ GPT) มีเอาต์พุตหลายตัว ทำให้ weight ชั้นซ่อนหนึ่งเส้น ส่งผลถึง loss ผ่าน หลายทางพร้อมกัน ตรงนี้แหละที่คนงง widget นี้จะพาไล่ที่มาของสมการ w ใหม่ทีละบรรทัด บนเครือข่าย 2 อินพุต → 2 ซ่อน → 2 เอาต์พุต ด้วยตัวเลขชุดคลาสสิกจากบทความในตำนานของ Matt Mazur — A Step by Step Backpropagation Example (กดไล่ทีละขั้นแล้วเทียบกับบทความได้เลขตรงกันทุกตัว)

🔬 กด ▶ ไล่ทีละขั้น — เส้นทางกฎลูกโซ่จะ "สว่างขึ้น" บนภาพ พร้อมแทนตัวเลขจริงให้เห็นครบ

ภาพจะไฮไลต์เฉพาะเส้นที่กำลังคิด (ที่เหลือหรี่ลง) · แผงล่างโชว์สูตรสัญลักษณ์ → แทนเลข → ผลลัพธ์ · ไฮไลต์สี: อินพุต · โหนดซ่อน · เอาต์พุต · สัญญาณผิดพลาด δ

🎬 เครื่องคลี่กฎลูกโซ่ — ดู ∂E/∂w "แตกตัว" ออกมาทีละท่อน แล้วตัวกลางหักกันเอง

ทำไม ∂Etotal/∂w5 ถึงกลายเป็น ผลคูณ 3 ท่อน ได้? กด ▶ ดูอนิเมชันการแทรกตัวกลาง (out_o1, net_o1) เข้าไป แล้วเห็นว่ามันหักล้างกันเองเหมือนเศษส่วน (telescoping) เหลือกลับมาเท่าเดิมเป๊ะ — เลือกได้ 2 กรณี: w5 (เอาต์พุต) และ w1 (ชั้นซ่อน — แตกสองทาง!)

💡 w6, w7, w8 (ปุ่มสีเทา) แสดงแบบสัญลักษณ์ล้วน — โครงเดียวกับ w5 เป๊ะ ต่างแค่ว่าเชื่อมโหนดไหน · ส่วน w2, w3, w4 คลี่แบบเดียวกับ w1 (แตกสองทางเสมอ เพราะเป็น weight ชั้นซ่อน)

🌿 ต่อยอด: ครบทั้ง 4 เส้นชั้นซ่อน (w1–w4) — fork และ "สูตรร่วม" ของ δ

weight ชั้นซ่อนซับซ้อนกว่าเพราะโหนดซ่อนหนึ่งตัวแตกไปเลี้ยงทั้ง 2 เอาต์พุต (fork) · เลือก w1–w4 แล้วดูว่าแต่ละเส้นคลี่อย่างไร (สัญลักษณ์ล้วน) · ไฮไลต์ท้ายสุด: ทำไม w1 กับ w2 ถึงใช้ δ ตัวเดียวกัน — ความลับที่ทำให้ backprop เร็ว

🏔️ ลึกขึ้นอีกขั้น: 2 ชั้นซ่อน — เมื่อ δ กลายเป็น "ลูกโซ่เวียนเกิด" (recursion)

เพิ่มชั้นซ่อนเป็น 2 ชั้น (อินพุต → ซ่อน A → ซ่อน B → เอาต์พุต) · จุดที่เข้าใจยากคือ δ ของชั้นลึกต้องใช้ δ ของชั้นที่อยู่ถัดไปทางเอาต์พุต — เป็น "สูตรเดียวกันวนซ้ำ" (recursion) · กด ▶ ดูคลื่น δ ไหลย้อนทีละชั้น

⚙️ แล้วของจริงเขาทำยังไง? — จาก "ไล่มือ" สู่ framework (PyTorch/TensorFlow)

ทุกอย่างที่เราไล่มือมา — δ, chain rule, fork, telescoping, recursion 2 ชั้น — framework ทำให้อัตโนมัติด้วยกลไกชื่อ autograd (reverse-mode automatic differentiation) · คุณแทบไม่ต้องหาสูตรอนุพันธ์เองเลยในชีวิตจริง แต่ที่เรียนมาทำให้รู้ว่าข้างในมันทำอะไร

🔗 จับคู่: สิ่งที่เราทำมือ ↔ คำสั่ง framework

ขั้นที่เราทำด้วยมือPyTorch ทำให้อัตโนมัติ
นิยาม weight + เครือข่ายnn.Linear, nn.Sequential — ตั้ง requires_grad=True ให้เอง
Forward คำนวณ ŷ (เก็บค่ากลางทาง)yh = model(x) — บันทึกทุก operation เป็น computational graph
loss = ½(y−ŷ)²loss = ((y-yh)**2).sum()*0.5 หรือ nn.MSELoss()
Backward: δ ทุกชั้น (chain rule, fork, recursion)loss.backward() — บรรทัดเดียว ไล่ย้อนทั้งกราฟให้หมด
∂L/∂w = δ × aเก็บไว้ใน param.grad อัตโนมัติ
w ← w − η·gradoptimizer.step()
ล้าง grad ก่อนรอบใหม่optimizer.zero_grad() (ลืมแล้ว grad จะสะสมผิด!)
🧠 autograd ทำงานยังไง: ตอน forward framework จดทุก operation ลง computational graph (บวก คูณ σ ...) · พอเรียก loss.backward() มันก็เดินกราฟย้อนกลับ ใช้ chain rule ทีละ node — ก็คือ δ-recursion + telescoping ที่เราไล่มือเป๊ะ เพียงแต่ทำให้ทุก operation อัตโนมัติ · เรียกว่า reverse-mode automatic differentiation

💻 โค้ดจริง — เครือข่าย 2 ชั้นซ่อน (เหมือนการ์ดบน) รันใน Colab ได้เลย

import torch, torch.nn as nn

# เครือข่าย 2 → 2 → 2 → 2 (2 ชั้นซ่อน) เหมือนที่เราไล่มือ
model = nn.Sequential(
    nn.Linear(2, 2), nn.Sigmoid(),   # ชั้นซ่อน A
    nn.Linear(2, 2), nn.Sigmoid(),   # ชั้นซ่อน B
    nn.Linear(2, 2), nn.Sigmoid(),   # เอาต์พุต
)
x = torch.tensor([[0.05, 0.10]])
y = torch.tensor([[0.01, 0.99]])
opt = torch.optim.SGD(model.parameters(), lr=0.5)   # η = 0.5

for step in range(10000):
    opt.zero_grad()            # ล้าง grad เก่า (สำคัญ!)
    yh = model(x)              # FORWARD — สร้าง computational graph
    loss = 0.5 * ((y - yh)**2).sum()  # L = ½Σ(y−ŷ)²
    loss.backward()            # BACKWARD — autograd หา δ ทุกชั้นให้เอง!
    opt.step()                 # w ← w − η·grad

print(loss.item())           # loss ลดลงเรื่อยๆ จนเกือบ 0

# แอบดู gradient ที่ autograd คำนวณ = ∂L/∂w ที่เราทำมือ:
for name, p in model.named_parameters():
    print(name, p.grad)

👉 loss.backward() บรรทัดเดียว = ทุกอย่างในหน้านี้ (δ_o → δ_B → δ_A, fork, telescoping) · เราไม่ต้องหาสูตรอนุพันธ์เองเลย — แต่ที่เรียนมาทำให้ debug เป็น (เช่นเห็น p.grad เป็น 0 ก็รู้ว่าเจอ vanishing gradient)

framework อื่นก็หลักเดียวกัน:
• TensorFlow / Keras — ระดับสูงใช้ model.compile() + model.fit() ซ่อนลูปทั้งหมด · ระดับล่างคุมเองด้วย tf.GradientTape() (จดกราฟแบบเดียวกับ autograd)
• JAX — ฟังก์ชันล้วน ใช้ grad(loss_fn) แปลงฟังก์ชัน loss ให้กลายเป็นฟังก์ชันหา gradient ทันที
• ทั้งหมดตั้งอยู่บนหลัก reverse-mode autodiff = chain rule ย้อนกลับที่คุณเพิ่งไล่มือทั้งหน้านี้
สรุปสูตรแบบพกพา — จำ 3 บรรทัดนี้ทำ backprop ได้ทั้งเครือข่าย

1) สัญญาณผิดพลาดของเอาต์พุต: δo = (ŷ − y) · ŷ(1−ŷ)  — รวม "พลาดเท่าไหร่" กับ "ความชัน sigmoid" ไว้ในตัวเดียว

2) ส่งย้อนหาโหนดซ่อน (จุดที่คนพลาด): δh = (Σ ของ δ ปลายทางทุกตัว × weight ของเส้นนั้น) · ah(1−ah)  — โหนดซ่อนต้องรวมความผิดจากทุกเอาต์พุตที่มันไปถึง

3) ความชันของ weight ใดๆ: ∂E/∂w = δ(ปลายเส้น) · a(ต้นเส้น)  แล้ว  wใหม่ = wเดิม − η·∂E/∂w — สูตรเดียวใช้ได้ทุกเส้นในทุกชั้น นี่คือทั้งหมดของ backprop

ส่วนที่ 6 — แบบฝึกหัด แบบฝึกหัด

ข้อ 1 (ทำในคาบ) — backprop มือเปล่าบนเครือข่ายจิ๋วสุด 1-1-1

เครือข่าย: x → [w₁, b₁, sigmoid] → h → [w₂, b₂, sigmoid] → ŷ · loss = ½(y−ŷ)²
กำหนด x = 1, เป้า y = 1, w₁ = 0.5, b₁ = 0, w₂ = 0.5, b₂ = 0, η = 1
จงทำครบวงจร 1 รอบ: forward → loss → backward → update (ปัด 4 ตำแหน่ง)

ดูเฉลย (ละเอียดทุกบรรทัด) Forward:
z_h = w₁x + b₁ = 0.5·1 + 0 = 0.5 → a_h = σ(0.5) = 0.6225
z_o = w₂a_h + b₂ = 0.5·0.6225 + 0 = 0.3112 → ŷ = σ(0.3112) = 0.5772
Loss: L = ½(1 − 0.5772)² = ½(0.4228)² = 0.0894
Backward:
δ_o = (ŷ−y)·ŷ(1−ŷ) = (−0.4228)(0.5772)(0.4228) = −0.1032
∂L/∂w₂ = δ_o·a_h = (−0.1032)(0.6225) = −0.0642 · ∂L/∂b₂ = δ_o = −0.1032
δ_h = δ_o·w₂·a_h(1−a_h) = (−0.1032)(0.5)(0.6225)(0.3775) = −0.0121
∂L/∂w₁ = δ_h·x = −0.0121 · ∂L/∂b₁ = δ_h = −0.0121
Update (η=1): w₂ = 0.5+0.0642 = 0.5642 · b₂ = 0.1032 · w₁ = 0.5121 · b₁ = 0.0121
ตรวจ: forward ใหม่ได้ ŷ ≈ 0.6124 → L ใหม่ ≈ 0.0751 < 0.0894 ✓ loss ลดจริง — เครื่อง "เรียนรู้" แล้วหนึ่งก้าว
ข้อ 2 (ทำในคาบ) — gradient descent เลขล้วน

ให้ L(w) = (w−3)² + 1 เริ่มที่ w = 0, η = 0.25 · จงคำนวณ w หลังก้าวที่ 1, 2, 3 (สูตร: dL/dw = 2(w−3))

ดูเฉลย ก้าว 1: slope = 2(0−3) = −6 → w = 0 − 0.25(−6) = 1.5
ก้าว 2: slope = 2(1.5−3) = −3 → w = 1.5 + 0.75 = 2.25
ก้าว 3: slope = 2(2.25−3) = −1.5 → w = 2.25 + 0.375 = 2.625
สังเกต: ระยะห่างจากคำตอบ (w*=3) หดครึ่งหนึ่งทุกก้าว (3 → 1.5 → 0.75 → 0.375) เพราะตัวคูณ = 1−2η = 0.5
ข้อ 3 (ทำในคาบ) — ทำไมเครือข่ายลึกเรียนช้า

σ′(z) = a(1−a) มีค่าสูงสุดเท่าไหร่ ที่ a เท่าไหร่? ถ้าเครือข่ายลึก 10 ชั้น สัญญาณ gradient ที่ไหลถึงชั้นแรกจะเหลืออย่างมากที่สุดกี่เท่าของต้นทาง? แล้ว ReLU (ความชัน = 1 เมื่อ z>0) ช่วยอย่างไร?

ดูเฉลย สูงสุด = 0.25 ที่ a = 0.5 (พิสูจน์: a(1−a) เป็นพาราโบลาคว่ำ จุดยอดที่ a=½)
ผ่าน 10 ชั้น ≤ 0.25¹⁰ = 1/1,048,576 ≈ 9.5×10⁻⁷ — ชั้นแรกแทบไม่ขยับ = vanishing gradient
ReLU ให้ความชัน 1 ในโซนบวก → คูณกี่ชั้นก็ไม่หด (ถ้าเส้นทางยัง active) จึงเทรนเครือข่ายลึกมากๆ ได้จริง
ข้อ 4 (การบ้าน) — η ใหญ่เกินแล้วระเบิดจริงไหม

โจทย์เดิมข้อ 2 (L(w)=(w−3)²+1 เริ่ม w=0) แต่ใช้ η = 1.5 · จงคำนวณ w หลังก้าวที่ 1–3 แล้วแสดงว่าระยะห่างจาก w*=3 "โตขึ้น" ทุกก้าวด้วยตัวคูณเท่าไหร่ พร้อมอธิบายเงื่อนไขของ η ที่ทำให้ลู่เข้า (การบ้าน — ไม่มีเฉลย)

ข้อ 5 (การบ้าน) — โยงสู่ GPT

GPT ถูกเทรนด้วยลูป forward → loss → backward → update เดียวกับที่เราทำมือวันนี้ จงระบุว่าในกรณี GPT: (ก) "อินพุต" และ "เป้าหมาย y" คืออะไร (ข) loss ใช้ตัวไหน (ใบ้: cross-entropy กับ softmax) (ค) อะไรคือสิ่งที่ต่างจากแล็บของเราบ้าง (ขนาด/ฮาร์ดแวร์/จำนวนรอบ) (การบ้าน — ไม่มีเฉลย)

ส่วนที่ 7 — ขั้นบันไดของสัปดาห์นี้ ขั้นบันได

สัปดาห์นี้พาเราเข้าใกล้ LLM อย่างไร:
วันนี้คุณคำนวณ อัลกอริทึมเดียวกันเป๊ะกับที่ใช้เทรน GPT ด้วยมือเปล่า — ไม่มีเวทมนตร์อื่นซ่อนอยู่
ต่างกันแค่สเกล: แล็บของเรามี weight 9 ตัว เทรนทีละ 1 ตัวอย่าง — GPT มี weight หลายแสนล้านตัว ไล่ backprop วงจรเดิมนี้ ล้านๆ รอบ บนข้อความทั้งอินเทอร์เน็ต ด้วย GPU หลายพันตัวนานหลายเดือน · ตัว loss ที่ GPT ลด ก็คือ "ทายคำถัดไปพลาดแค่ไหน" (cross-entropy) — ทุกครั้งที่ ChatGPT ตอบเก่ง นั่นคือผลของ gradient descent ที่คุณเพิ่งกดเล่นเมื่อครู่
สัปดาห์หน้า: ได้เครื่องมือเรียนรู้ครบมือแล้ว — เอาไปสอนเครื่องให้ "มองเห็นภาพ" กับ CNN สถาปัตยกรรมที่เข้าใจธรรมชาติของรูปภาพ
