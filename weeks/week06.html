Decision Tree & Random Forest: เครื่องที่ "ถามทีละคำถาม"

KNN วัดระยะ, K-means จับกลุ่ม — คราวนี้เปลี่ยนแนวคิดใหม่หมด: โมเดลที่ถามคำถาม yes/no ทีละข้อเหมือนเกม 20 คำถาม จนได้คำตอบ · ข้อดีคือ อ่านกฎออกด้วยตา (interpretable) ไม่ต้องใช้คณิตเลย

เป้าหมายสัปดาห์นี้ — เรียนจบแล้วนักศึกษาต้อง:
  1. อธิบายได้ว่า Decision Tree คือชุดคำถาม if/else ต่อกัน (root → กิ่ง → ใบ) และอ่านกฎออกได้ ต่างจาก KNN ที่วัดระยะ
  2. เข้าใจว่าเครื่องเลือก "คำถามที่ดีที่สุด" ยังไง — คำนวณ Gini impurity ของแต่ละจุดตัดแล้วเลือกตัวที่ทำให้กลุ่มย่อย "บริสุทธิ์" ที่สุด
  3. มองออกว่าเส้นแบ่งของต้นไม้เป็น ขั้นบันไดตั้งฉากแกน และต้นไม้ลึกเกินไปจะ overfit (จำ noise)
  4. อธิบาย Random Forest ได้: ปลูกต้นไม้หลายต้นบนข้อมูลสุ่มย่อย แล้วโหวต → ลด overfit และเส้นแบ่งเนียนขึ้น
  5. รู้ว่าทำไม tree/forest ยังครอง ข้อมูลตาราง (tabular) และงาน Kaggle ส่วนใหญ่ ขณะที่ neural network ครองภาพ/ข้อความ

ส่วนที่ 1 — แนวคิด: เล่นเกม "20 คำถาม" กับข้อมูล แนวคิด

เคยเล่นเกมทายของไหม? อีกฝ่ายคิดคำตอบไว้ในใจ เราถามได้แต่คำถามที่ตอบ "ใช่/ไม่ใช่" — "มีชีวิตไหม?" "ตัวใหญ่กว่าแมวไหม?" ค่อย ๆ ตัดตัวเลือกลงจนเหลือคำตอบเดียว · Decision Tree ก็คิดแบบนี้เป๊ะ ๆ มันเรียงคำถาม yes/no ต่อกันเป็น "ต้นไม้" จากบนลงล่างจนถึง "ใบ" ที่เป็นคำตอบ

ต่างจากสัปดาห์ก่อนชัดเจน: KNN ตอบโดยวัดระยะไปหาเพื่อนบ้าน (ต้องคิดเลขระยะทาง) — แต่ Decision Tree ตอบด้วยกฎที่อ่านออก ("ถ้าฝนตก และลมแรง → ไม่เล่น") ไม่มีคณิตอะไรเลย มีแค่การเปรียบเทียบทีละเงื่อนไข นี่คือเหตุผลที่มันเป็นโมเดลที่ อธิบายให้คนฟังรู้เรื่อง (interpretable) มากที่สุดตัวหนึ่ง

ตัวอย่างจริง: "วันนี้ควรออกไปเล่นกีฬาไหม?" — เครื่องเรียนจากสถิติที่ผ่านมาแล้วสร้างต้นไม้นี้ ลองไล่ตามลูกศรจากบนลงล่างดู:

ส่วนที่ 3 — เครื่องเลือก "จุดตัดที่ดีที่สุด" ยังไง? (Gini) ทีละขั้น

Gini คืออะไร — วัด "ความปนเปื้อน" ของกล่องยังไง

Gini impurity ตอบคำถามเดียว: กล่องข้อมูลนี้ "ปนกัน" มากแค่ไหน? · เข้าใจง่ายสุดผ่านเกมนี้ — หยิบลูกบอลจากกล่องมา 1 ลูกแบบสุ่ม แล้วเดาสีมันด้วยการจับฉลากตามสัดส่วนสีในกล่อง · โอกาสที่จะเดาผิด นั่นแหละคือค่า Gini · กล่องสีเดียวล้วน → เดาไม่มีวันผิด (G=0 บริสุทธิ์) · กล่องครึ่ง-ครึ่ง → เดาผิดบ่อยสุด (G=0.5 ปนสุด)

4

เส้นโค้ง Gini: พุ่งสูงสุด 0.5 ตอนสีปนกันครึ่ง-ครึ่ง แล้วดิ่งลง 0 เมื่อกล่องเหลือสีเดียว · จุด ● = กล่องตอนนี้

เกมพิสูจน์: "โอกาสเดาผิด = Gini"

กด ▶ ให้เครื่องเล่นเกม "หยิบ 1 ลูก แล้วเดาสีตามสัดส่วน" ซ้ำเร็ว ๆ แล้วดู อัตราเดาผิดจริง ค่อย ๆ ลู่เข้าหาค่า Gini ที่คำนวณไว้เป๊ะ

ที่มาของตัวเลข: ลองทุกจุดตัด แล้ววัด "ความปนเปื้อน" (Gini)

มีจุด 10 จุดเรียงบนแกนเดียว (ฟีเจอร์ x) · แดง 4 จุด, น้ำเงิน 6 จุด · เครื่องจะลอง "เส้นตัด" ทุกช่องว่างระหว่างจุด แล้วสำหรับแต่ละเส้น คำนวณ Gini impurity ของกลุ่มซ้าย/ขวา แล้วเฉลี่ยแบบถ่วงน้ำหนัก · ยิ่ง Gini น้อย = กลุ่มยิ่งบริสุทธิ์ (เหลือสีเดียว) · เลื่อนดูทีละเส้นตัด แล้วดูตัวเลขคำนวณสด ๆ:

สูตร Gini ของกลุ่มหนึ่ง: G = 1 − (pแดง² + pน้ำเงิน²)   โดย p = สัดส่วนของแต่ละสีในกลุ่ม
กลุ่มสีเดียวล้วน → G = 0 (บริสุทธิ์สุด) · ครึ่ง-ครึ่ง → G = 0.5 (ปนสุด) · คะแนนรวมของเส้นตัด = ถ่วงน้ำหนักตามจำนวนจุด: (nซ้าย/N)·Gซ้าย + (nขวา/N)·Gขวา

เส้นที่ให้คะแนนรวมต่ำสุด = คำถามที่ดีที่สุดที่โหนดนี้จะถาม · ต้นไม้จริงในส่วนที่ 2 ก็เลือกด้วยวิธีนี้เป๊ะ ๆ เพียงแต่ทำกับทั้งสองแกน (x และ y) แล้วเลือกอันที่ดีที่สุด

เช็กความเข้าใจเร็ว ๆ

กลุ่มหนึ่งมี แดง 3 จุด กับ น้ำเงิน 1 จุด (รวม 4) · Gini ของกลุ่มนี้เท่าไหร่?

ดูเฉลย pแดง=3/4=0.75, pน้ำเงิน=1/4=0.25 → G = 1 − (0.75² + 0.25²) = 1 − (0.5625 + 0.0625) = 1 − 0.625 = 0.375

# 1) ชุดข้อมูลของเล่น "พระจันทร์คู่" (โค้ง แยกด้วยเส้นตรงไม่ได้)
X, y = make_moons(n_samples=400, noise=0.30, random_state=42)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.3, random_state=0)

# 2) ต้นไม้ต้นเดียว — ลอง max_depth หลายค่า ดูอาการ overfit
for d in [1, 3, 6, None]:      # None = ลึกไม่จำกัด → จำ noise
    tree = DecisionTreeClassifier(max_depth=d, random_state=0).fit(X_tr, y_tr)
    tr = accuracy_score(y_tr, tree.predict(X_tr))
    te = accuracy_score(y_te, tree.predict(X_te))
    print(f"depth={str(d):>4} | train={tr:.2f}  test={te:.2f}")

# 3) วาดโครงต้นไม้ — อ่านกฎ if/else ได้ด้วยตา
tree = DecisionTreeClassifier(max_depth=3, random_state=0).fit(X_tr, y_tr)
plt.figure(figsize=(11, 6))
plot_tree(tree, filled=True, feature_names=["x", "y"], class_names=["A", "B"])
plt.show()

# 4) Random Forest — ปลูกหลายต้นแล้วโหวต (ลด overfit เอง)
rf = RandomForestClassifier(n_estimators=200, random_state=0).fit(X_tr, y_tr)
print("Forest  | test =", accuracy_score(y_te, rf.predict(X_te)))

# 5) ฟีเจอร์ไหนสำคัญ? — ป่าบอกได้เลย
print("feature_importances_ =", rf.feature_importances_)

ลองเปลี่ยน noise ให้สูงขึ้น หรือปรับ max_depth / n_estimators แล้วสังเกตช่องว่าง train–test — จะเห็นเรื่องเดียวกับเวทีด้านบนเป๊ะ ๆ

ส่วนที่ 10 — แบบฝึกหัด แบบฝึกหัด

ข้อ 1 (ทำในคาบ) — คำนวณ Gini ด้วยมือ

โหนดหนึ่งมีจุด 6 จุด: แดง 2, น้ำเงิน 4 · (ก) หา Gini ของโหนดนี้ · (ข) ถ้าตัดแล้วได้ ซ้าย = {แดง 2} และ ขวา = {น้ำเงิน 4} คะแนน Gini รวม (ถ่วงน้ำหนัก) เป็นเท่าไหร่ · (ค) การตัดนี้ดีไหม เพราะอะไร

ดูเฉลย (ก) pแดง=2/6=0.333, pน้ำเงิน=4/6=0.667 → G = 1 − (0.333² + 0.667²) = 1 − (0.111 + 0.444) = 0.444
(ข) ซ้าย {แดง 2} บริสุทธิ์ → G=0 · ขวา {น้ำเงิน 4} บริสุทธิ์ → G=0 · รวม = (2/6)(0) + (4/6)(0) = 0
(ค) ดีมาก — Gini ลดจาก 0.444 เหลือ 0 (ทั้งสองฝั่งเหลือสีเดียวล้วน) แปลว่าคำถามนี้แยกคลาสได้สมบูรณ์
ข้อ 2 (ทำในคาบ) — ทำไมต้นไม้ลึกถึง overfit

อธิบายด้วยคำพูดตัวเอง: ทำไมต้นไม้ที่ปล่อยให้ลึกไม่จำกัดถึงได้ train accuracy = 100% แต่ test accuracy กลับแย่? และการจำกัด max_depth ช่วยยังไง

ดูเฉลย ต้นไม้ลึกไม่จำกัดจะแตกกิ่งจนแต่ละใบเหลือจุดฝึกไม่กี่จุด (หรือจุดเดียว) — มัน "จำ" ตำแหน่งของทุกจุดฝึก รวมทั้ง noise/ข้อยกเว้น ทำให้ทายข้อมูลฝึกถูกหมด แต่กฎที่ห่อ noise พวกนั้นไม่ใช่แนวโน้มจริง พอเจอข้อมูลใหม่จึงพลาด · จำกัด max_depth บังคับให้ต้นไม้จับได้แต่ "รูปแบบใหญ่ ๆ" ที่ generalize ได้ ไม่ลงไปจำรายละเอียดจุกจิก
ข้อ 3 (ทำในคาบ) — ป่าดีกว่าต้นเดียวเพราะอะไร

Random Forest ปลูกต้นไม้หลายต้นแล้วโหวต · จง (ก) บอกว่าต้นไม้แต่ละต้นเห็นข้อมูล/ฟีเจอร์ต่างกันยังไง (bootstrap + สุ่มฟีเจอร์) · (ข) อธิบายว่าทำไมการ "เฉลี่ยหลายต้น" ถึงลด overfit

ดูเฉลย (ก) แต่ละต้นฝึกบน bootstrap sample (สุ่มหยิบจุดจากชุดฝึกแบบใส่คืน ได้ชุดข้อมูลคนละหน้าตา) และที่แต่ละ split ยังสุ่มพิจารณาเฉพาะบางฟีเจอร์ → ต้นไม้แต่ละต้นจึงหลากหลาย ไม่เหมือนกัน
(ข) ต้นไม้แต่ละต้น overfit คนละแบบ = ผิดคนละจุด (error สุ่ม ๆ ไม่สัมพันธ์กัน) · เมื่อโหวตเสียงข้างมาก ความผิดสุ่มหักล้างกันไป เหลือแต่สัญญาณจริงที่ทุกต้นเห็นตรงกัน → variance ลด เส้นแบ่งเนียนขึ้น ทน noise ขึ้น
ข้อ 4 (การบ้าน) — Tree/Forest ปะทะ KNN และ Neural Net

(ก) เปรียบเทียบ Decision Tree กับ KNN (เรื่อง KNN): แต่ละตัวตัดสินใจด้วยอะไร, ตัวไหน "อ่านกฎออก" · (ข) ทำไม tree/forest จึงยังครองงานข้อมูลตาราง (tabular) และการแข่ง Kaggle ขณะที่ neural network (เรื่อง Neural Network) ครองงานภาพ/ข้อความ (การบ้าน — ไม่มีเฉลย)

ส่วนที่ 11 — ขั้นบันไดของสัปดาห์นี้ ขั้นบันได

เราเดินทางถึงไหนแล้ว:
KNN / K-means (วัดระยะในปริภูมิ) → Decision Tree (เปลี่ยนเป็น "กฎ if/else ที่อ่านออก" — โมเดลคลาสสิกที่ interpretable ที่สุด) → Random Forest (รวมหลายต้นแบบ ensemble ให้แม่นและทน noise) — นี่คือสายพันธุ์ tree-based ที่ยังครองงาน ข้อมูลตาราง (tabular) และเวที Kaggle จนถึงทุกวันนี้ (คู่กับ XGBoost/LightGBM)
สัปดาห์หน้า (เรื่อง SVM): SVM — เปลี่ยนมุมอีกครั้ง ไปหาเส้นแบ่งที่ "เว้นระยะขอบ (margin) ให้กว้างที่สุด" แทนการซอยเป็นขั้นบันได · แล้ว Neural Network จะพาไปสู่โมเดลที่เรียนรู้ฟีเจอร์เอง ซึ่งครองงานภาพ/ข้อความและเป็นบล็อกก่อสร้างของ LLM
บทเรียนติดตัว: ความสมดุลระหว่าง "จำเก่ง (overfit)" กับ "generalize ได้" ที่เห็นในต้นไม้วันนี้ คือหัวใจเดียวกับที่เราจะเจอตลอดทั้งวิชา (เต็ม ๆ ใน Overfitting & Regularization)
